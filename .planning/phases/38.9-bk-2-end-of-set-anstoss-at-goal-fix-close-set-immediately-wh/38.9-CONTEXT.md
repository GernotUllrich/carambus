# Phase 38.9 — BK-2 End-of-Set Anstoss-at-Goal Fix

## Why this phase exists

Discovered during phase 38.7 UAT TR-Ctl test. In a BK-2 untied final inning, when Anstoss reaches `balls_goal` in inning ≥ 2 the set should close immediately (the Erste-Aufnahme-Gate failed — Nachstoss is no longer eligible). Currently the set stays open silently; ScoreEngine switches `active_player` unconditionally; Player B plays an unauthorized inning; the legacy parity gate finally closes the set without ever showing the red "Nachstoß" banner.

## User-stated symptom (verbatim)

> "Spiel BK-2; Spieler macht in der zweiten Aufnahme aus. Jetzt sollte das Spiel eigentlich beendet sein. Springt aber zu Spieler B ohne die rote Message Nachstoß. Nach wechsel beendet das Spiel"

## Root cause (already diagnosed)

Three-way semantic mismatch between `follow_up?`, `end_of_set?`, and `terminate_inning_data`:

- `TableMonitor#follow_up?` correctly enforces the Erste-Aufnahme-Gate at `app/models/table_monitor.rb:1188-1190` (Nachstoß only when Anstoss reaches goal in inning 1) → suppresses the banner correctly
- `TableMonitor#end_of_set?` (lines 1456-1521) is missing the matching "close immediately" branch: Plan 02's D-02 branch requires +1 innings lag; legacy karambol branch needs innings parity or `allow_follow_up=false` (BK-2 forces it true) → no branch fires → set stays open
- `ScoreEngine#terminate_inning_data` at line 1306 switches `active_player` unconditionally → Player B plays unauthorized inning

**Pre-existing latent defect** introduced in commit `79328663` (2026-04-28, before phase 38.7). NOT a regression from plans 02/05/13.

**Full diagnosis:** `.planning/debug/bk2-nachstoss-banner-missing.md`
