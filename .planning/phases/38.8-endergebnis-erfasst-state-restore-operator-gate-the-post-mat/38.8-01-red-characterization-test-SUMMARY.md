## Decisions Made

- **Test assertion targets `tm.state == "final_match_score"` literal string** — the AASM state column stores the symbol-as-string (`app/models/table_monitor.rb:343` confirms). Avoids pinning a stringified-or-symbol ambiguity.
- **Starting state set to `:set_over` via update_columns** — Branch C (the buggy training-rematch path) is only reachable when ResultRecorder enters from `:set_over`. From `:playing` it lands in `:set_over` and returns before reaching the bug. This forced the test design to skip the natural `:playing → :set_over` transition and go straight to Branch C.
- **Scores untied (100 vs 60), not tied** — tied scores would trigger `tiebreak_pick_pending?` in the phase 38.7 guard (result_recorder.rb:467-470) which short-circuits before Branch C's auto-rematch. The bug lives in the auto-rematch block, so the test must NOT trigger the tiebreak guard.
- **Sanity precondition assertions included** — `assert_equal "set_over", @tm.state` before the call; `assert_nil @tm.tournament_monitor`; `assert_not_nil @tm.game`. These distinguish a genuine contract failure (`Expected: "final_match_score" / Actual: "playing"`) from a setup crash if a future maintainer accidentally breaks the fixture shape.
