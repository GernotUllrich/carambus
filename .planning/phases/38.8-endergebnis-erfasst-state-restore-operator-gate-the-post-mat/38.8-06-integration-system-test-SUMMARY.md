## Decisions Made

- **Per-instance @seqno_counter (Rule 3 — blocking-issue auto-fix).** Plan-prescribed code used `seqno: 1` hard-coded in build_training_tm. The Game model has `validates :seqno, uniqueness: { scope: %i[tournament_id gname], message: "Duplicate game in group", allow_nil: true }` (game.rb:416-420). Training games have nil tournament_id and nil gname, so the scope tuple is (nil, nil), and the validation IS enforced (allow_nil only refers to seqno itself). Successive calls in the SC-4 loop collided on the second iteration with `ActiveRecord::RecordInvalid: Gültigkeitsprüfung ist fehlgeschlagen: Seqno Duplicate game in group`. Fix: `@seqno_counter ||= 0; @seqno_counter += 1; Game.create!(seqno: @seqno_counter, ...)`. The fix is documented inline at the helper's leading comment block. Cross-instance leakage is not an issue because the teardown destroys all created Games + each test gets a fresh test instance.
- **standardrb fixes applied to new code only.** Pre-existing baseline standardrb offenses in test/models/table_monitor_test.rb (16 offenses on lines 95/108-109/123-147/161-171/204 — Layout/ParameterAlignment, Layout/ArgumentAlignment, Style/SingleLineMethods) NOT touched per plan instruction "Do NOT modify any existing test in this file." My new code introduced 3 NEW offenses: hash-literal-brace spaces on lines 256/272/273; all fixed inline before commit. New system test had 3 NEW offenses (RescueModifier line 149, ExtraSpacing line 186 ×2); all fixed before commit. Final standardrb status on the new file: clean (0 offenses). On the modified file: 16 baseline offenses preserved (1 of which is `EmptyLinesAroundClassBody` on line 303, the same offense that was on baseline line 248 before my +55-line insertion shifted it; not introduced by me).
- **Cross-discipline loop scope: karambol + BK-2 only.** Pool/Snooker fixtures are not reachable in the test DB without a substantial fixture-graph investment. Plan permits a `skip` annotation; the practical equivalent (loop over only the available disciplines) achieves the same regression-locking outcome. Future work can extend the loop to Pool/Snooker without touching surrounding logic. The test name remains "cross-discipline" because the contract being locked is "every discipline available lands in :final_match_score" — not "specifically these N disciplines."
- **TableMonitor.find singleton override approach.** The reflex's `start_rematch` invokes `@table_monitor = TableMonitor.find(element.dataset.id)`; we need it to return our test-local `tm` instance with the spy stubs attached. Approach: define a class-level singleton `find` method that returns `tm` for any input, run the reflex, then restore via `TableMonitor.singleton_class.send(:remove_method, :find)` inside an `ensure` block. The `rescue` is defensive (the method may have been removed already by an earlier test or not present at all on Ruby's singleton class hierarchy). Pattern matches Phase 38.7 Plan 05 stub-and-restore convention.
- **Spy via define_singleton_method, not Mocha/MiniTest mocks.** Same pattern as Phase 38.8-04 Test 1 (TM advance_tournament_round_if_present); same pattern as Task 1 of this plan. Rationale: define_singleton_method is dependency-free, requires no `require "mocha/minitest"` (which is not in this project), and mirrors the existing convention in result_processor_test.rb. Spies record into instance variables (`@revert_spy = true`); assertions use `instance_variable_get`.
