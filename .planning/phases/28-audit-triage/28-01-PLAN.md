**bin/check-docs-translations.rb (AUDIT-02):**
- Class `DocsTranslationChecker` with `DOCS_ROOT = Pathname.new(__dir__).join('..', 'docs').expand_path`
- Glob `**/*.de.md` and `**/*.en.md` under DOCS_ROOT
- Strip double extension to get base names: `Pathname.new(f).sub_ext('').sub_ext('').to_s` (strip `.md` then `.de`/`.en`)
- Compute set difference: `missing_en = de_bases - en_bases`, `missing_de = en_bases - de_bases`
- Accept `--nav-only` flag: when present, parse `mkdocs.yml` nav entries (use `YAML.load_file` from stdlib) to build set of nav-referenced base paths, then filter results to only those base paths. The nav YAML is nested arrays/hashes; walk recursively extracting all string values (the `.md` file paths), strip `.md` extension to get base name, and check if `{base}.de.md` and `{base}.en.md` both exist
- Accept `--exclude-archives` flag: exclude paths containing `/archive/`, `/obsolete/`, `/internal/`, `/studies/`, `/changelog/`
- Output format: one line per finding: `MISSING_EN: docs/path/to/file.en.md` or `MISSING_DE: docs/path/to/file.de.md`
- Print summary: total de files, total en files, missing en count, missing de count
- Exit 0 if no gaps, exit 1 if any gaps found
- Make executable: `chmod +x`

**bin/check-docs-coderef.rb (supports AUDIT-01, per D-03/D-04):**
- Class `DocsCoderefChecker` with same DOCS_ROOT and APP_ROOT (`Pathname.new(__dir__).join('..').expand_path`)
- **Step 1 — Build stale identifier list from git diff:**
  - Run `git diff --diff-filter=D --name-only v1.0 v5.0 -- app/ lib/` via backticks to get deleted files
  - Run `git diff --diff-filter=R --name-status v1.0 v5.0 -- app/ lib/` via backticks to get renamed files
  - For each deleted/renamed file, convert basename to CamelCase: split on `_`, capitalize each segment, join. Example: `umb_scraper_v2.rb` becomes `UmbScraperV2` (handle trailing digits: `v2` -> `V2`). Also keep the snake_case filename without extension as an identifier (e.g., `tournament_monitor_support`)
  - Store these as the `stale_identifiers` set (both CamelCase and snake_case forms)
- **Step 2 — Build live identifier set:**
  - Glob `{app,lib}/**/*.rb` files
  - For each, extract the basename without `.rb`, convert to CamelCase, add to `live_names` set
- **Step 3 — Scan docs for stale references (full-text, not code-fence-only):**
  - Glob `docs/**/*.md`, exclude `/archive/`, `/obsolete/`, `/internal/` paths
  - For each doc file, read content and search for any `stale_identifiers` member (use `Regexp.union(stale_identifiers.to_a)` for efficient matching)
  - Record: `{ file:, line:, identifier:, context: line.strip }` for each match
  - **Why full-text scanning is safe here (diverges from RESEARCH.md code-fence recommendation):** The research Pitfall 1 warns about false positives from broad CamelCase extraction — scanning all CamelCase tokens in prose produces hundreds of hits (YouTube, GitHub, etc.). This script avoids that pitfall entirely by inverting the approach: instead of extracting tokens from docs and checking against the codebase, it builds a tiny, specific set of identifiers from `git diff` (currently just `UmbScraperV2`, `TournamentMonitorSupport`, `tournament_monitor_support`) and searches for those exact strings. With only 3 search terms derived from git history, false positives are impossible — these identifiers are unique to the project's deleted/renamed classes. Full-text scanning is preferable because stale references appear in prose and headings (e.g., "The `TournamentMonitorSupport` module handles..."), not only in code fences.
- Accept `--exclude-archives` flag (same exclude dirs)
- Accept `--json` flag: output findings as JSON array instead of text
- Output format (text): `STALE_REF: docs/path/file.md:42 — UmbScraperV2`
- Print summary: files scanned, stale identifiers checked, findings count
- Exit 0 if no stale refs found, exit 1 if any found
- Make executable: `chmod +x`

**lib/tasks/mkdocs.rake — append mkdocs:check task (AUDIT-03):**

Inside the existing `namespace :mkdocs do` block, before the closing `end`, add:

```ruby
  desc "Validate MkDocs documentation — strict mode, exits non-zero on any warning (CI-ready)"
  task check: :environment do
    unless system("which mkdocs > /dev/null 2>&1")
      puts "Error: mkdocs is not installed. Please install it first:"
      puts "  pip install mkdocs-material mkdocs-static-i18n pymdown-extensions"
      exit 1
    end

    # Build to temp dir to avoid polluting project with site/ artifacts
    tmp_dir = "/tmp/mkdocs-check-#{Process.pid}"
    puts "Running mkdocs build --strict (output to #{tmp_dir})..."
    success = system("mkdocs build --strict --site-dir #{tmp_dir} 2>&1")

    # Cleanup temp build artifacts
    FileUtils.rm_rf(tmp_dir) if Dir.exist?(tmp_dir)

    unless success
      puts "\nError: mkdocs build failed with warnings or errors."
      puts "Fix all warnings before proceeding."
      exit 1
    end

    puts "\nDocumentation validation passed — zero warnings."
  end
```
