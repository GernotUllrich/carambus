NOTE: Writing a plain `Hash` value into an `ActionController::Parameters` object SILENTLY
turns the value into a Parameters too on read in some Rails versions, but at minimum the
outer `p` remains a Parameters object. Even when bk2_options reads back as a Hash, the
*outer* p is still Parameters, so when forwarded as `options: p` to GameSetup,
`HashWithIndifferentAccess.new(p)` triggers the unsafe conversion path that raises
`ActionController::UnfilteredParameters` on the unpermitted nested keys.
