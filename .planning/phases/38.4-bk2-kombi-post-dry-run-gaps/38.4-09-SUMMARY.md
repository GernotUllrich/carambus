## Decisions Made

- **Outer wrapper REMOVED, not retained:** The plan was explicit (`<interfaces>` lines 113-127) that the original `<div x-show="..." class="col-span-6 mt-4 space-y-3">` is the GAP-02 anti-pattern — it consumed one parent-grid cell and stacked 4 rows inside it via `space-y-3`, breaking visual alignment with sibling Spieler/Disziplin rows. The replacement gives 4 row pairs (8 sibling divs) that share the parent `grid-cols-6 gap-3 items-center` alignment.
- **Bespoke template-x-for chosen over reusing `_radio_select` partial for the Punkt-Ziel row:** The plan accepted both approaches as valid (line 567). The partial expects a static Ruby-side `values:` array; for Punkt-Ziel we needed Alpine-reactive `bk_ballziel_choices` driven by `bk_selected_form`. Extending the partial signature to accept Alpine getters would complicate every other consumer; bespoke inline template-x-for is the right call here.
- **Stale 38.4-06 comment header (lines 314-316) removed during Task 1:** Applied Rule 1 (bug fix — stale comments are misleading documentation). Replaced with a single Plan 38.4-09 header that cross-references the GAP-02 invariant and the hidden-input separation.
- **Regex anchor fix in T-DZ-max-touchbutton + T-SP-max-touchbutton tests:** The plan-supplied regex `is_bk_dz_configurable.*?is_bk_sp_configurable` matched first against the JavaScript getter definitions at lines 224-225 (where the two getter names appear adjacently), not against the row-block usages further down. Applied Rule 1 inline: anchor on `x-show="[^"]*is_bk_dz_configurable` (the row's HTML attribute) instead of the bare getter name.
