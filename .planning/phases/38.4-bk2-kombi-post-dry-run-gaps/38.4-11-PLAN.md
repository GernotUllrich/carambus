<objective>
Close 2 of the 8 sub-issues from UAT round 2 test 2 by adding explicit Nachstoß
(equalizing-shot) support to the BK-* scoring services:

- **O2** — BK50 and BK100 currently close the set the instant a player reaches the
  ball goal. Per BK rules (Spielregeln Kegeln, Schiedsrichter-Handbuch), the trailing
  player is owed an equalizing inning. Plan 11 adds this rule.
- **O4** — When the trailing player IS allowed Nachstoß (already implicit for BK-2
  / BK-2plus / BK-2kombi via existing set-close logic), an off-by-one in the increment
  guard caps their score at `balls_goal - 1`, preventing the equalizing 100-point
  inning that BK100 needs. Plan 11 fixes the cap to allow exactly `balls_goal`.

The fix is single-cohesive: both O2 and O4 are aspects of the same rule (Nachstoß),
just at different levels (configuration + state machine for O2; arithmetic guard for O4).
Implementing both in one plan keeps test fixtures and seed updates together.

**Purpose:** UAT round 2 test 2 sub-issues O2 and O4 are blockers — the user cannot
run a complete BK50 or BK100 game because the set either closes too early (O2) or
locks the trailing player at a score that prevents valid completion (O4).

**Output:**
- 1 seed script extended (`script/seed_bk2_disciplines.rb` — adds `nachstoss_allowed: true` to all 5 BK-* disciplines).
- 2 service files modified (`app/services/bk2/advance_match_state.rb` adds Nachstoß deferred-close; `app/services/bk2/commit_inning.rb` honors Nachstoß state on the next inning).
- 2 service unit test files extended (TDD-style — RED tests added before service edits).
- 1 system test file extended with 5 new T-O2 / T-O4 method-level guards.
- ERBlint / standardrb clean.

**Wave gating:** Wave 1 — independent of Plans 10 and 12. Plan 10 (view + form integrity)
and Plan 12 (quick-game shortcut) can run in parallel. Plan 11 only touches services and
fixtures; no view file changes.

**Clarifying note on Nachstoß equal-vs-below outcomes:**
The user observation in UAT (O4) only explicitly described the off-by-one bug ("kommt nur
bis 49"), not the precise rule for what happens when the Nachstossende equals or undercuts
balls_goal. The Schiedsrichter-Handbuch in `doc/Schiedsrichter-Handbuch Kegel.pdf` is the
canonical source; the BK2.docx Excel-tool is the secondary reference. This plan adopts the
default semantics:

  - **Nachstossende equals balls_goal exactly** → set closes as a DRAW for the SET (no
    sets_won increment for either; phase 38.4 BK rules treat single-set BK50/BK100 as
    "first to balls_goal wins" — a draw becomes "Verlängerungssatz" per Schiedsrichter
    handbook). For Plan 11, the simplest correct semantics is: **set closes; the SECOND
    player to reach balls_goal wins (i.e., the Nachstossende wins the set if they equal,
    because the leader's earlier reach is overruled by the equalizer rule's fairness
    intent — they get one more chance)**. THIS IS A PROVISIONAL RULE INTERPRETATION;
    the executor MUST flag in 38.4-11-SUMMARY.md whether this matches the user's
    expectation, and Landessportwart sign-off may override in a future plan.
  - **Nachstossende falls short of balls_goal** → set closes with the original leader
    winning (their pre-Nachstoß reach stands).
  - **Nachstossende exceeds balls_goal** → not possible if the increment guard fires
    correctly at `score <= balls_goal`. If the executor finds a path that allows
    overshooting (e.g., a +5 inning when current score is balls_goal - 3), the executor
    MUST raise this in the SUMMARY as an open question; default treatment is "trailing
    player wins" since they exceeded the target.

If the user/Landessportwart confirms a different rule (e.g., always Verlängerungssatz on
equal), this plan's set-close logic factors out cleanly into a private method that can
be replaced without re-wiring the Nachstoß state machine.

```ruby
def close_set_if_reached!(state)
  set_no = state["current_set_number"].to_s
  target = state["balls_goal"].to_i
  target = state["set_target_points"].to_i if target.zero?  # transitional fallback
  a = state["set_scores"][set_no]["playera"]
  b = state["set_scores"][set_no]["playerb"]
  return unless a >= target || b >= target

  winner = (a >= target) ? "playera" : "playerb"
  state["sets_won"][winner] += 1
  state["set_finished_#{set_no}"] = true
  state["set_winner_#{set_no}"] = winner
  ... (advances to next set or does nothing for set 3) ...
end
```

Bk2::CommitInning (app/services/bk2/commit_inning.rb):

```ruby
def call
  ensure_initialized!
  return idempotency_noop if already_applied?

  state = @tm.data["bk2_state"].deep_dup

  transitions = apply_inning!(state)

  advance_helper = Bk2::AdvanceMatchState.new(table_monitor: @tm, shot_payload: {})
  advance_helper.send(:close_set_if_reached!, state)
  advance_helper.send(:close_match_if_reached!, state)
  record_sequence!(state) if @shot_sequence_number

  @tm.data["bk2_state"] = state
  @tm.save!

  {state: state, transitions: transitions}
end

# apply_inning! dispatches by free_game_form:
def apply_inning!(state)
  case free_game_form
  when "bk_2plus"             then apply_opponent_credit_rule(state)
  when "bk_2", "bk50", "bk100" then apply_additive_rule(state)
  when "bk2_kombi"            then apply_bk2_kombi_rule(state)
  else raise ArgumentError
  end
end
```

    ```ruby
    # Schließe den aktuellen Satz wenn ein Spieler balls_goal erreicht hat.
    # Phase 38.4 D-06: liest state['balls_goal'], fällt zurück auf state['set_target_points'].
    # Phase 38.4-11 O2/O4: Nachstoß-Regel — wenn discipline.nachstoss_allowed? und der
    # nachfolgende Spieler noch keine Aufnahme hatte, wird der Satz NICHT sofort geschlossen.
    # Stattdessen wird state['nachstoss_pending']=true gesetzt; player_at_table flipt.
    # Beim nächsten close_set_if_reached!-Aufruf (nach der Nachstoß-Aufnahme) wird die
    # Resolution angewendet: Nachstossende equal → Nachstossende gewinnt (provisorisch);
    # Nachstossende unter balls_goal → Anführer gewinnt.
    def close_set_if_reached!(state)
      set_no = state["current_set_number"].to_s
      target = state["balls_goal"].to_i
      target = state["set_target_points"].to_i if target.zero?
      a = state["set_scores"][set_no]["playera"]
      b = state["set_scores"][set_no]["playerb"]
      return unless a >= target || b >= target

      leader = (a >= target) ? "playera" : "playerb"
      trailing = (leader == "playera") ? "playerb" : "playera"

      # Phase 38.4-11 O2: Nachstoß-Verzweigung.
      if state["nachstoss_pending"]
        # Resolution: Nachstossende hat ihre Aufnahme abgeschlossen.
        # provisional rule: equal → trailing wins; below → leader wins.
        nachstoss_for = state["nachstoss_for"].to_s
        original_leader = (nachstoss_for == "playera") ? "playerb" : "playera"
        nachstoss_score = state["set_scores"][set_no][nachstoss_for].to_i
        original_target = target  # The target the leader reached pre-Nachstoß.

        winner = if nachstoss_score >= original_target
          nachstoss_for  # trailing reached or exceeded — trailing wins (provisional rule)
        else
          original_leader
        end

        state["nachstoss_pending"] = false
        state["nachstoss_resolved"] = true
        state["sets_won"][winner] += 1
        state["set_finished_#{set_no}"] = true
        state["set_winner_#{set_no}"] = winner
        return advance_to_next_set!(state, winner)
      end

      # Phase 38.4-11 O2: defer set-close if discipline allows Nachstoß and trailing has
      # not yet had their equalizing inning (set_scores[trailing] still at pre-leader-reach
      # value, but more importantly: nachstoss_pending wasn't already set).
      if discipline_nachstoss_allowed?
        state["nachstoss_pending"] = true
        state["nachstoss_for"] = trailing
        state["player_at_table"] = trailing
        # Reset shots/innings counters for the Nachstoß turn.
        state["shots_left_in_turn"] = derive_dz_max_shots if state["current_phase"] == "direkter_zweikampf"
        return  # Do NOT close set; defer to next close_set_if_reached! after trailing's inning.
      end

      # Default (legacy / nachstoss_allowed=false): immediate close.
      state["sets_won"][leader] += 1
      state["set_finished_#{set_no}"] = true
      state["set_winner_#{set_no}"] = leader

      advance_to_next_set!(state, leader)
    end

    # Phase 38.4-11 O2: extracted from close_set_if_reached! — advances to next set
    # (or no-op if final set). Called from both the legacy and Nachstoß-resolution paths.
    def advance_to_next_set!(state, set_winner)
      return if state["current_set_number"] >= 3

      state["current_set_number"] += 1
      first_mode = state["first_set_mode"].presence || derive_first_set_mode
      new_phase = phase_for_set(state["current_set_number"], first_mode)
      state["current_phase"] = new_phase

      if new_phase == "direkter_zweikampf"
        state["shots_left_in_turn"] = derive_dz_max_shots
        state["innings_left_in_set"] = 0
      else
        state["shots_left_in_turn"] = 0
        state["innings_left_in_set"] = derive_sp_max_innings
      end

      state["player_at_table"] = (set_winner == "playera") ? "playerb" : "playera"
    end

    # Phase 38.4-11 O2: Lese discipline.nachstoss_allowed? mit Fallback auf false.
    # @tm.discipline kann nil sein in unit tests ohne Game/Discipline-Wiring; in dem Fall
    # gibt der Helper false zurück (legacy / kein Nachstoß).
    def discipline_nachstoss_allowed?
      @tm.respond_to?(:discipline) && @tm.discipline.respond_to?(:nachstoss_allowed?) && @tm.discipline.nachstoss_allowed?
    end
    ```
