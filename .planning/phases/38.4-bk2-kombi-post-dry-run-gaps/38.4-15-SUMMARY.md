## Decisions Made

- **Capybara DOM test → ERB-compilation test (I-15-01 methodology shift):** the original Plan 15 draft used `visit table_monitor_path(@tm)`, but that route renders the table_monitor scoreboard, not the BK-* detail view. The detail view is rendered via `LocationsController#show` with `sb_state=free_game_detail`, which requires Location/Table/Club/Player fixture chains not present in `Bk2ScoreboardTest`. Rather than introduce a fixture chain just for one assertion, the test compiles the ERB template directly via `ActionView::Template::Handlers::ERB.new` and asserts against the resulting Ruby source. This catches the I-15-01 concern (class string surviving compilation rather than being stripped as a comment) without browser overhead. Documented as a methodology shift here per the plan's `output:` section requirement.
- **Sibling spacer (col-span-6 h-4) over mt-N on the partial:** chose to insert a standalone spacer div rather than adding a margin class to the MEHRSATZ partial because the partial would have to grow a `mt:` local parameter and the spacer cleanly fills one row of the parent `grid-cols-6`. The `gap-3` on the parent grid is insufficient when the row above is variable-height.
- **cols=4 BK-* / cols=5 non-BK Aufnahmebegrenzung:** kept the two sibling partial calls (one per `show:` predicate) rather than collapsing into one call with a dynamic `cols:`. The `_radio_select` partial doesn't have an Alpine-template escape hatch for dynamic `cols:`, and the asymmetric button counts make a single dynamic call awkward.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] T-P2-bk-variante-row-cols-5-rendered Capybara test fails — `visit table_monitor_path(@tm)` doesn't render the BK-* detail view**
- **Found during:** Task 3 (running the new tests)
- **Issue:** The Plan 15 draft Capybara test used `visit table_monitor_path(@tm)` (mirroring Plan 10's existing T-shootout-* and T3 setup pattern) and asserted against `page.html`. But `table_monitor_path` routes to `TableMonitorsController#show`, which renders the scoreboard view (not `scoreboard_free_game_karambol_new.html.erb`). The latter view is rendered via `LocationsController#show` with `sb_state=free_game_detail`. Test failed with `Expected /grid grid-cols-5 gap-4/ to match "..."` against the table-monitor page HTML.
- **Fix:** Replaced the Capybara/DOM approach with an ERB-compilation render via `ActionView::Template::Handlers::ERB.new.call(template, raw_erb)`. This compiles the ERB to Ruby source — class strings inside `<%# %>` comments are stripped, plain HTML class strings survive as string literals. Catches the same I-15-01 concern (class survives compile) without needing the complex Location/Table/Club/Player fixture chain that visiting the actual rendered route would require.
- **Files modified:** `test/system/bk2_scoreboard_test.rb` (the single test method body)
- **Verification:** `bin/rails test test/system/bk2_scoreboard_test.rb -n "/T-P2-bk-variante-row-cols-5-rendered/"` → 1 run, 4 assertions, 0 failures, 0 errors. All 4 T-O6/T-P2 tests PASS together.
- **Committed in:** `88fabf5d` (Task 3 commit — single edit; original draft never landed in a separate commit because the fix was applied during the same task before the commit)
