# Phase 38.4 Plan 05: Rename Bk2Kombi→Bk2, BK-family dispatcher, balls_goal, Open-Q-2 resolution

Hard-rename `Bk2Kombi::*` → `Bk2::*` throughout, add 5-way scoring dispatcher in `Bk2::CommitInning`, align `Bk2::AdvanceMatchState` with D-06 `balls_goal` read, and resolve RESEARCH Open Q 2 for `result_recorder.rb`.

## Objective

Close I7 per D-10, D-11, D-12, D-13. The renamed `Bk2::CommitInning` is now the single entry point for all 5 BK-* disciplines, branching internally on `discipline.data[:free_game_form]`. `result_recorder.rb` dispatches all 5 forms to `Bk2::AdvanceMatchState` (Open Q 2 resolved). `Bk2::AdvanceMatchState` initializes state with `balls_goal` from `tournament_monitor.balls_goal` (D-06), writing a transitional `set_target_points` fallback for in-flight games.

## What Was Built

### Namespace rename (Task 1)

Three service files moved from `app/services/bk2_kombi/` to `app/services/bk2/` via `git mv`. Module declarations changed from `module Bk2Kombi` to `module Bk2`. Three test files moved from `test/services/bk2_kombi/` to `test/services/bk2/`. Integration test renamed from `bk2_kombi_dispatch_integration_test.rb` to `bk2_dispatch_integration_test.rb`. All `Bk2Kombi::` references purged from `app/`, `test/`, and `config/` — verified at zero.

### BK-family dispatcher in `Bk2::CommitInning`

Added `apply_inning!` dispatcher with 5-way `case form` statement:
- `bk_2plus` → `apply_opponent_credit_rule` (negative inning → abs credited to opponent)
- `bk_2`, `bk50`, `bk100` → `apply_additive_rule` (sign-preserving, negative stays on player)
- `bk2_kombi` → `apply_bk2_kombi_rule` (phase-dependent: DZ=opponent-credit, SP=additive)
- unknown → `ArgumentError` (no silent mis-routing)

Added `free_game_form` reader: reads `discipline.data` (JSON parse with rescue) first, falls back to `@tm.data["free_game_form"]` for the quick-game path.

### `Bk2::AdvanceMatchState` D-06 alignment

Added `derive_balls_goal`: reads `@tm.tournament_monitor&.balls_goal.to_i` (positive?), then `bk2_options["set_target_points"]`, then `DEFAULT_SET_TARGET`. Updated `init_state_if_missing!` to call `derive_balls_goal` and write both `"balls_goal"` and legacy `"set_target_points"` (transitional). Updated `close_set_if_reached!`: reads `state["balls_goal"]` first, falls back to `state["set_target_points"]` if zero.

### Open Q 2 resolution in `result_recorder.rb`

Extended the dispatch condition from `== "bk2_kombi"` to `Discipline::BK2_FREE_GAME_FORMS.include?(@tm.data["free_game_form"])`. Service call renamed from `Bk2Kombi::AdvanceMatchState` to `Bk2::AdvanceMatchState`. CommitInning is NOT used here — result_recorder.rb is the tournament-game persistence path; AdvanceMatchState is the correct target.

### View predicate expansion

`_scoreboard.html.erb` and `_player_score_panel.html.erb`: `is_bk2` expanded from single-value `"bk2_kombi"` to 5-value array check. Plan 05 is the sole owner of this predicate expansion.

### Reflex updates

`table_monitor_reflex.rb`: 4 `Bk2Kombi::` call sites updated to `Bk2::`. Three BK-family init guards updated from `== "bk2_kombi"` to `bk_family?` (the 5-value predicate from Plan 04). `bk2_kombi_commit_if_active` guard similarly extended. Added `bk_family?` private helper using `Discipline::BK2_FREE_GAME_FORMS.include?`.

The `bk2_first_set_mode` write guard intentionally preserved as `== "bk2_kombi"` — only BK-2kombi has the 4-button shootout mode selector (D-17).
