### Discipline Data Model (I1)

- **D-04:** **New central Discipline records via carambus_api** for 4 new
  disciplines (BK50, BK100, BK-2, BK-2plus). BK-2kombi
  (`Discipline.find(107)`) already exists — kept. Each new record carries:
  - `data[:free_game_form]` matching its internal key
    (`bk50`, `bk100`, `bk_2`, `bk_2plus`, `bk2_kombi`)
  - `data[:ballziel_choices]` (see D-07)
  - Standard Discipline attributes (name, category, etc.)
  - Records sync to all local servers via the existing Version infrastructure
    (once D-05 unblocks it).

- **D-05:** **Fix sync-version-yaml-load-json-collision bug INSIDE 38.4 as
  the first plan.** The open todo (2026-04-23,
  `.planning/todos/pending/2026-04-23-sync-version-yaml-load-json-collision.md`)
  blocks D-04: `Version#update_from_carambus_api` (version.rb:330/338) calls
  `YAML.load` on JSON-encoded text columns and returns a Hash, making
  `update_columns` fail on text columns. Without this fix, 4 new central
  Discipline records cannot propagate cleanly to BCW / phat / master without
  manual unprotected=true writes. Fix it once, reap benefits for all future
  discipline sync.

- **D-06:** **Ballziel stored in tournament-level `balls_goal`** (Karambol-
  style), NOT in `bk2_options[:set_target_points]`. Per-set target =
  `balls_goal`; match target = `sets_to_win`.
  - **BK-2 / BK-2plus:** single-set match. `sets_to_win=1` (or absent),
    `balls_goal` = chosen ball target.
  - **BK-2kombi:** best-of-3. `sets_to_win=2`, `sets_to_play=3`,
    `balls_goal` = per-set ball target (50/60/70).
  - **BK50 / BK100:** single-set match. `balls_goal` = 50 / 100 (fixed).
  - `Bk2::CommitInning#close_set_if_reached?` reads `balls_goal` (not
    `bk2_options[:set_target_points]`).
  - `bk2_options[:set_target_points]` is **retired** — planner decides whether
    to delete entirely or keep dormant for one milestone with a deprecation
    comment + migration for in-flight BK2-Kombi tournaments.
  - **I9 closure:** this migration replaces the buggy Alpine empty-string →
    `to_i` → 0 path. Regression test = set closes at configured balls_goal
    across all 5 BK-\* disciplines.

- **D-07:** **Allowed Ballziel ranges on `discipline.data[:ballziel_choices]`:**
  - BK50 = `[50]` (fixed, read-only UI)
  - BK100 = `[100]` (fixed, read-only UI)
  - BK-2 = `[50, 60, 70, 80, 90, 100]`
  - BK-2plus = `[50, 60, 70, 80, 90, 100]`
  - BK-2kombi = `[50, 60, 70]`
  - Detail-view renders radio/dropdown by reading this list from the selected
    discipline. Server-side CLAMP on `TableMonitorsController` permit whitelist.

### Scoring Generalization (I7)

- **D-10:** **Service rename: `Bk2Kombi::CommitInning` → `Bk2::CommitInning`**
  (same for `Bk2Kombi::AdvanceMatchState` → `Bk2::AdvanceMatchState` per D-13).
  All 5 BK-\* disciplines route through `Bk2::CommitInning`. Karambol score
  accumulator is no longer reached for BK-\*. Branching logic:
  - `free_game_form == "bk_2plus"`: opponent-credit rule (negative inning total
    → absolute value to opponent's set score, player unchanged).
  - `free_game_form == "bk_2"` / `"bk50"` / `"bk100"`: sign-preserving
    additive (inning total added to player's own set score, including
    negatives; opponent unchanged).
  - `free_game_form == "bk2_kombi"`: phase-dependent —
    `current_phase == "direkter_zweikampf"` → opponent-credit (BK-2plus
    behaviour); `current_phase == "serienspiel"` → additive (BK-2 behaviour).
    This corrects the previous generic DZ-phase coding.

- **D-11:** **Dispatcher lives inside the service.** `TableMonitorReflex`
  always calls `Bk2::CommitInning.call(...)` for any BK-\* discipline.
  `Bk2::CommitInning` reads `table_monitor.discipline.data[:free_game_form]`
  (+ `bk2_state["current_phase"]` for bk2_kombi) and branches. Reflex stays
  thin. Mirrors the state-reading pattern already in 38.3's
  `Bk2Kombi::CommitInning`.

- **D-12:** **BK-2/BK50/BK100 allow negative inning totals.** No opponent
  credit. Player's set score may go negative. Consistent with 38.3 D-12 for
  the Serienspiel phase of BK-2kombi. Negative-tolerant display already landed
  in 38.3 Plan 03 (`_player_score_panel.html.erb`).

- **D-13:** **Also rename `Bk2Kombi::AdvanceMatchState` → `Bk2::AdvanceMatchState`.**
  Single namespace for BK-\* services. Generalizes set-close logic per D-06
  (`balls_goal` as per-set target). Update all call sites and test files.
  Dormant `Bk2Kombi::ScoreShot` → `Bk2::ScoreShot` (rename only; still unused
  on live path).
