## Architecture Notes

### x-effect approach for SP-max mirror (O7)

The `_radio_select` partial has no Alpine-template x-for escape hatch for dynamic button sets. The cleanest approach was two separate partial calls sharing `varname: "innings"`. The `x-effect` attribute on the x-data wrapper div mirrors `innings → bk2_sp_max_innings` reactively whenever `innings` changes while `is_bk_family && !is_bk_fixed_goal` is true. This runs without any @change wiring on the partial calls themselves — Alpine re-evaluates the x-effect whenever its reactive dependencies change.

```html
x-effect="if (is_bk_family && !is_bk_fixed_goal) { bk2_sp_max_innings = parseInt(innings) || 5 }"
```

### BK-Variante 8-column grid (O6)

The plan spec noted that if the CSS auto-flow `grid-cols-8` with 5 buttons didn't produce flush-right alignment, a `col-span-3 col-start-6` spacer might be needed. The actual result with plain `grid-cols-8` places the 5 buttons naturally in cols 1-5; cols 6-8 stay empty. This matches the visual real estate occupied by the +/- correction controls on surrounding rows. Manual UAT (user post-merge) should confirm the right-flush appearance.
