Close 2 round-4 gaps (P1, P3) — both share a single root cause: the BK-* quick-start
path bypasses the new clamp logic when (a) the local Discipline record for the BK-*
discipline does not exist (or has empty `ballziel_choices`) on this server, OR (b) a
future regression accidentally drops `:balls_goal` from the slice list. Both manifest
as `bk2_state.balls_goal == 50` regardless of the button's declared `balls_goal=100`
(P1: BK100) or `balls_goal=70` (P3: BK-2kombi 2/5/70+NS).

**Root cause analysis (per investigation):**

1. The `_quick_game_buttons.html.erb` partial DOES emit `balls_goal` as a top-level
   hidden field (line 141: `hidden_field_tag :balls_goal, button['balls_goal'] || 50`).
2. The controller's slice list at line 73-88 INCLUDES `:balls_goal` (line 74 explicit;
   also re-listed at line 84 with explicit Phase 38.4-04 D-06 comment). The slice list
   ALSO includes `:balls_goal_a` and `:balls_goal_b` (line 74 — confirmed by reading
   the controller source 2026-04-25 for I-13-01 verification). These three fields
   form the fallback chain.
3. The controller's clamp logic at line 415-416:
   ```ruby
   discipline = Discipline.find_by(name: p[:discipline_a].to_s)
   allowed = discipline&.ballziel_choices.presence || [50]
   ```
   When `Discipline.find_by(name: "BK100")` returns nil (new BK-* records seeded only
   on carambus_api master, NOT yet propagated to carambus_bcw via Version sync per
   STATE.md "Sync bug — sync-version-yaml-load-json-collision"), `allowed = [50]`,
   `requested = 100`, `[50].include?(100) ? 100 : [50].first → 50`. Bug confirmed.
4. Same path for P3: `Discipline.find_by(name: "BK2-Kombi")` returns the local id-107
   record. If its `data['ballziel_choices']` is missing or wrong (Plan 11 seed not yet
   re-run on BCW after Plan 11's changes), `allowed.presence || [50] → [50]`.

**Fix strategy (defense-in-depth):**

A. Add a `BK_FAMILY_BALLZIEL_FALLBACK` constant in the controller carrying the SAME
   values as `script/seed_bk2_disciplines.rb`. Use it when `discipline&.ballziel_choices`
   is empty/nil. The seed remains canonical; the constant is a safety net for the API
   sync gap.
B. Make `clamp_bk_family_params!` also read `balls_goal_a`/`balls_goal_b` as fallbacks
   for `balls_goal` (the partial emits all three; some legacy callers may emit only
   `balls_goal_a`/`balls_goal_b`). This is a low-risk widening — the value is still
   intersected against `allowed`. Slice-list precondition VERIFIED (I-13-01): both
   fields are in the controller permit/slice list at line 74.
