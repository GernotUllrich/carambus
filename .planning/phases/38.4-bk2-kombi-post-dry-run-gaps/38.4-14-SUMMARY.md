# Phase 38.4 Plan 14: Route :goal_reached through Bk2::CommitInning Summary

**One-liner:** TableMonitor#add_n_balls and #set_n_balls now dispatch BK-family-with-nachstoss disciplines through Bk2::CommitInning (Option B name-based lookup) — closes round-4 UAT gap P4 ("kommt nur bis 49" + ProtokollEditor write-rejection), incidentally clears the 4 deferred T8-T11 errors as a side-effect.

## Outcome

- **P4 closure** — TableMonitor#add_n_balls and #set_n_balls now route `:goal_reached` through `Bk2::CommitInning` for BK-family disciplines with `nachstoss_allowed?`. Plan 11's `nachstoss_pending` deferred-close machinery is engaged on both the +1 button path AND the ProtokollEditor write path (where the user reported the bug).
- **Option B preservation** — `TableMonitor#discipline` STILL returns the String `data["playera"]["discipline"]`. 15+ legacy callers (Eurokegel/Biathlon/Pool variants/8-9-10-Ball/14.1-endlos comparisons + `ProtocolPdf` display) are UNAFFECTED. The deferred T8-T11 caller-migration phase remains an explicit future-phase decision, not an accidental change.
- **Bk2 service layer hardened** — Both `Bk2::CommitInning#free_game_form` and `Bk2::AdvanceMatchState#discipline_nachstoss_allowed?` now accept either an AR Discipline record or a String name (defensive `disc = Discipline.find_by(name: disc) if disc.is_a?(String)`). Required for Option B's dispatch to fire end-to-end without the previous `define_singleton_method` test-only stub.
- **Incidental side-benefit** — The 4 pre-existing T8-T11 errors documented in `38.4-VERIFICATION.md:38-43` (`NoMethodError: undefined method 'data' for "BK2-Kombi":String at app/services/bk2/commit_inning.rb:108`) are now resolved as a side-effect of the Option B `String → AR` defensive conversion. `bk2_scoreboard_test.rb` runs 55 tests with 0 errors (previously 47 + 4 errors).

## Decisions Made

(See frontmatter `key-decisions` for full list.)

The most consequential ones during execution:

1. **Option B (name-based lookup) over Option A (TableMonitor#discipline returns AR record)** — Plan-text already specified Option B; round-4 iteration-2 had already chosen this. Option A would have rippled into 15+ legacy callers; Option B is a tightly scoped fix.
2. **Fixing CommitInning + AdvanceMatchState to handle String discipline** — When the integration test surfaced that the dispatch ran but Nachstoß never engaged, traced to `discipline_nachstoss_allowed?`'s `respond_to?(:nachstoss_allowed?)` returning false on String. Same pattern as CommitInning's `discipline.data` crash on String. Both fixes are Rule 1/2 (correctness) — the Option B dispatch must work end-to-end.
3. **Test seed pattern** — Score_engine and the change-collection machinery have hidden requirements that production code satisfies incrementally. Tests that seed in one shot need explicit scaffolding (balls_on_table, both players' innings arrays, suppress_broadcast during seed). Documented in the test setup comments so future plans can reference the pattern.

**5. [Rule 1 — Bug] `recompute_result` semantics in score_engine — plan seed value wrong**

- **Found during:** First T-P4-add-n-balls-routes test PASSED setup but `add_n_balls` did NOT return `:goal_reached`. Traced to score_engine's `to_play = balls_goal - (result + innings_redo_list[-1])`. Plan-text seed was `result=69, innings_redo_list=[69]` → `to_play = 70 - (69+69) = -68` → `n_balls(1) <= to_play(-68)` is false → no `:goal_reached`.
- **Fix:** `result=0, innings_redo_list=[balls_goal-1]` → `to_play = 70 - (0+69) = 1` → `add == to_play` → `:goal_reached`. The `result` field is recomputed from `innings_list.sum`, not `innings_redo_list[-1]`.
- **Files modified:** `test/services/bk2/advance_match_state_test.rb`
- **Documented in:** test setup comments + key-decisions.

## Bk_family_with_nachstoss? False Positives

The two-condition check (free_form ∈ BK2_FREE_GAME_FORMS AND `Discipline.find_by(name: …)&.nachstoss_allowed? == true`) makes false positives unlikely:

- A legacy TM with stale `data["free_game_form"]` would need that form to be in `BK2_FREE_GAME_FORMS` AND the named discipline to carry `nachstoss_allowed: true`. Post-Plan-16, only BK-2kombi (id 107) carries the flag.
- A deliberately spoofed `data["playera"]["discipline"]` String pointing at "BK-2kombi" would route through Bk2::CommitInning — but the score-button reflex already trust-bounds writes via Pundit policies on TableMonitor. Defense-in-depth via the rescue ArgumentError fallback in route_goal_reached_through_bk2_commit_inning. No new attack surface.
