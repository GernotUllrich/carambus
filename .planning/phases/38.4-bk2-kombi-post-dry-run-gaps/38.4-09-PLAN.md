<objective>
Convert the BK-* detail-view block at lines 317-390 of `scoreboard_free_game_karambol_new.html.erb`
from bare radios + select dropdown + plain number inputs into 4 touch-button rows that match
the rest of the karambol scoreboard family (no selectors, no dropdowns — only large labelled
buttons that fingers can hit).

**Purpose:** Test 3 of 38.4-HUMAN-UAT is a minor visual/UX defect — the conditional
visibility logic (DZ/SP) works correctly, but the *style* doesn't match. The BK-pro player
operates on a tablet with no mouse, no precision pointer, and the existing detail view sets
the user's expectation: "all parameter selection is touch-buttons." The Plan 38.4-06 block
broke that expectation by introducing radio chrome + a `<select>` element. This plan
realigns the input affordance.

**Output:**
- 1 view file replaced (lines 317-390 of scoreboard_free_game_karambol_new.html.erb).
- 1 system test file updated (selectors for 3 existing tests + 1 new T-Punktziel-row test).
- ERBlint clean on the changed file.
- Single x-data scope preserved (GAP-02 invariant).

**Wave gating:** This is Wave 2 — depends on Plan 38.4-08 (the start_game crash fix from
Wave 1). Plan 09 should not begin until 38.4-08 lands cleanly, because verifying the new
DOM at runtime requires being able to actually start a BK-* game (the system test that
flexes shootout selectors only succeeds after the crash is fixed, and the UAT re-run for
this plan exercises the same code path).
