## button_id Collision — Preventive Disambiguation

The button_id collision between the new "BK-2kombi 2/5/70+NS" entry and the existing "70/Best of 3" entry (both: discipline=BK2-Kombi, balls_goal=70) did NOT manifest in pre-Plan-12 manual testing because the pre-Plan-12 layout had only 3 buttons (50/60/70) — all unique by balls_goal. The disambiguation is **preventive**: without it, both 70-ball entries would generate the same `bk2_kombi_70` button_id, causing duplicate `id="quick_player_a_bk2_kombi_70"` hidden field IDs on the page, breaking the JavaScript player-selection wiring for the second button.

The fix (adding `label_suffix = button['label'] ? "...gsub..." : ""` before `button_id` construction) is minimal and covers any future same-discipline+same-balls_goal additions automatically.
