### Display-Invariant Comment Added

A code-comment invariant was added at the top of the `bk2_kombi:` sub-tree in both locale files, documenting:
- The internal mode keys (`direkter_zweikampf`, `serienspiel`) are stored in `bk2_state["current_phase"]` and `bk2_options["first_set_mode"]`
- Only VALUES may change, never the KEY PATHS
- Display mapping: direkter_zweikampf → "BK-2plus", serienspiel → "BK-2" (BK-pro + Landessportwart sign-off)

### Key Structure Decision

The plan's task description showed a hypothetical `.label` nested sub-key, but the actual YAML and all production code use `phase.direkter_zweikampf` as a flat string value (e.g. `t("table_monitor.bk2_kombi.phase.#{current_phase}")`). Per D-09, YAML key PATHS stay unchanged. The flat structure was preserved and only the value was renamed.
