# Phase 38.4 Plan 02: BK2 Fallback Delete Escape-Hatch Summary

## Objective

Close I8 by adding a Delete button to the BK2 "Spiel nicht initialisiert" fallback banner in `app/views/table_monitors/_show.html.erb`. A Schiri seeing the fallback page previously had no way to scrap a stuck TableMonitor from the scoreboard — they had to navigate to the admin list. This plan closes that gap.

## What Was Built

### Task 1: Add Delete button to BK2 fallback banner + DE/EN i18n keys

**Commit:** `884f6bf6`

**Files modified:**

1. `app/views/table_monitors/_show.html.erb` — inserted `button_to` block after the `uninitialized_banner_body` paragraph and before the closing `</div>` of the fallback banner, inside the `if table_monitor.bk2_state_uninitialized?` block (lines 106-110 after edit).

   The button:
   - Uses `button_to` (not `<a>`) — auto-wraps in a Rails CSRF-protected form (T-38.4-02-02 mitigated)
