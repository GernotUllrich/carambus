## Round-4 Iteration-2 BLOCKER 2 + I-16-02 + I-16-INFO-01 Closure Summary

**Round-4 iteration-2 BLOCKER 2 (LocalProtector silent rollback):**

- **Root cause:** `LocalProtector#disallow_saving_global_records` (after_save) raises `ActiveRecord::Rollback` when global records are mutated on local servers. AR silently swallows `Rollback` inside the surrounding transaction → `update!` returns `true` while the row is rolled back. The original `rescue ActiveRecord::RecordInvalid` block was UNREACHABLE — the rescue NEVER fired on local servers; the `cleared += 1` counter was incremented falsely.
- **Fix landed:** Replaced rescue-based pattern with EXPLICIT pre-check `if !ApplicationRecord.local_server? || rec.id >= MIN_LOCAL_ID` then `rec.update!(data: parsed.to_json)`. On master: pre-check passes → update! fires → PaperTrail records the change → Version sync propagates. On local servers: for global records (id < MIN_LOCAL_ID), pre-check skips with `skipped_local += 1` + an explanatory log. Local-DB cleanup flows from master via Version sync.
- **Verification:** T-P5-clear-nachstoss-migration-present asserts (a) `if !ApplicationRecord.local_server?` present, (b) `rec.update!(data:` present, (c) `rescue ActiveRecord::RecordInvalid` ABSENT, (d) `update_columns` ABSENT, (e) `skipped_local` counter present.

**I-16-02 (residual sync-race):**

- **Documented in migration source comments** (lines 26-36 of `db/migrate/20260425090000_clear_nachstoss_allowed_for_non_bk2_kombi.rb`): "If a stale Plan-11 payload arrives via Version sync AFTER the migration runs, the flag will REAPPEAR on that server until the seed re-runs on master AND the new flag-less payload propagates."
