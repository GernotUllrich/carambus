# Phase 38.4 Plan 01: Version.safe_parse — Sync YAML/JSON Collision Fix Summary

**One-liner:** Added `Version.safe_parse` and `Version.safe_parse_for_text_column` helpers to fix YAML.load returning Hash for JSON-encoded text columns in PaperTrail sync payloads, replacing all 4 buggy callsites and adding regression test coverage.

## What Was Built

### The Bug

`Version#update_from_carambus_api` had two code paths (Path A: update-invalid-record fallback; Path B: new-record creation) that called `YAML.load(args["data"])` and `YAML.load(args["remarks"])`. Because valid JSON is also valid YAML flow syntax, `YAML.load('{"free_game_form":"bk2_kombi"}')` returns a Ruby Hash instead of preserving the original JSON string. The subsequent `update_columns(args)` then rejects the Hash for a `text` column with "can't cast Hash". The error was swallowed by the surrounding `rescue StandardError => e` at version.rb, so the sync silently failed.

### The Fix

Two new class methods added to `app/models/version.rb`:

**`Version.safe_parse(raw)`** — General-purpose parse helper:
- Returns `raw` unchanged if blank
- If `raw.lstrip` starts with `{` or `[`, uses `JSON.parse` (returns Ruby object)
- Otherwise falls back to `YAML.load` for legacy Rails-serialized-hash columns
- Rescues `JSON::ParserError` / `Psych::Exception` / `ArgumentError` → returns raw input

**`Version.safe_parse_for_text_column(raw)`** — Text-column-safe wrapper:
- Returns `raw` unchanged if blank
- If `raw.lstrip` starts with `{` or `[` AND JSON is valid → returns `raw` string unchanged (text column gets text, no Hash cast)
- If YAML returns a Hash/Array → converts to JSON string via `.to_json` (text column always receives String)
- Falls back to raw on any parse failure

### Callsites Replaced (4 total)

| Event | Path | Old | New |
|-------|------|-----|-----|
| create | non-PartyCc data | `YAML.load(args["data"])` | `Version.safe_parse_for_text_column(args["data"])` |

- **Found during:** Task 1, integration test implementation
- **Issue:** The plan spec called `Version.update_from_carambus_api(payload)` passing an Array directly, but the method signature is `opts = {}` (a Hash). Passing an Array caused `TypeError: no implicit conversion of Symbol into Integer` at `opts[:update_tournament_from_cc]`.
- **Fix:** Rewrote integration test to use `stub_request` (WebMock, already active in test env) to stub the internal HTTP GET call, then call `Version.update_from_carambus_api({})`. Used `update` event (not `create`) to exercise the Path A/B `h["object"]` fallback branch — the exact code path with the bug.
- **Files modified:** test/models/version_test.rb
- **Impact:** More faithful integration test — exercises the actual HTTP fetch path, not a bypassed internal call.
