# Phase 38.3 Plan 04: Point-Entry Reflex Wire-up Summary

**One-liner:** Deleted deprecated shot-entry reflex+controller (D-23) and wired Karambol swap/opponent-panel gestures to `Bk2Kombi::CommitInning` via private helper `bk2_kombi_commit_if_active` in `next_step`/`key_a`/`key_b` — Route A, zero new Stimulus code.

## What Was Built

### Task 1 — Deletion audit (D-23)

Removed all traces of the deprecated shot-entry approach:

- `def bk2_kombi_submit_shot` (lines 984–1034 of `table_monitor_reflex.rb`) and its full docstring removed
- `app/javascript/controllers/bk2_kombi_shot_controller.js` deleted via `git rm`
- Verified no remaining live callers in `app/views/`, `app/javascript/`, or `config/` (only `test/system/bk2_kombi_scoreboard_test.rb` had stale references — deferred to Plan 38.3-07 rewrite per plan spec)

### Task 2 — Karambol reflex branching (D-14)

Added `bk2_kombi_commit_if_active(gesture_id:, clicked_player: nil)` as a private helper in the `private` section of `TableMonitorReflex`:

- **Security S1**: `player` read from `@table_monitor.data["bk2_state"]["player_at_table"]` — server-controlled JSONB, never client DOM
- **Security S2**: `free_game_form` gate reads from `@table_monitor.data["free_game_form"]` — server-controlled, not client attribute
- **D-14 clicked_player guard**: `key_a` passes `clicked_player: "playera"`, `key_b` passes `clicked_player: "playerb"`; helper returns false (no-op) when clicked player == player_at_table (tapping own panel), commits when opponent panel clicked
- **Idempotency**: generates fresh `SecureRandom.uuid` per commit for `shot_sequence_number` (T-38.3-04-04)
- **Inning reset**: after CommitInning, sets `innings_redo_list[-1] = 0` and appends `0` so next Aufnahme starts clean

Early branches added to:
- `next_step` — swap-icon click: commits + `save!` + returns before Karambol `reset_timer!`/`terminate_current_inning`
- `key_a` — Spieler-A panel click: commits if opponent, returns before Karambol path
- `key_b` — Spieler-B panel click: commits if opponent, returns before Karambol path

`add_n`, `minus_n`, `numbers` reflexes left completely unchanged.

### Task 3 — Bottom-bar verification (no code change)

Confirmed the existing Karambol bottom bar already renders for BK2-Kombi without code change:

- All four reflex wiring attributes present in `_scoreboard.html.erb`: `table-monitor#add_n`, `table-monitor#minus_n`, `table-monitor#next_step`, `table-monitor#numbers`
- BK2 TableMonitor found in dev DB (`id=50000009`, `playing?=true`) — bottom bar renders via existing `if fullscreen && table_monitor.playing?` guard
- No view-layer code changes required or made
