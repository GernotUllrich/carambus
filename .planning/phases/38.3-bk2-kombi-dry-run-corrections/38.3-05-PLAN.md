<objective>
Move the BK2-Kombi first_set_mode 4-button matrix from the detail-view (where Plan 38.2-02 put it) to the shootout-screen (D-18 / DR-01). The detail-view keeps only the generic game-configuration inputs (Plan 38.3-06 rewires it). The shootout is the correct home for the Anstoß × first-set-mode decision because it's the point-of-no-return where the match actually starts.

Implementation:
- `_shootout.html.erb` branches on `show_data["free_game_form"] == "bk2_kombi"`.
- BK2 branch renders 4 buttons in a 2×2 grid: Spieler A DZ, Spieler A SP, Spieler B DZ, Spieler B SP.
- Each button fires a reflex that BOTH (a) writes `bk2_options[:first_set_mode]` into @table_monitor.data AND (b) transitions via the existing Karambol shootout path (start_game for left/A player, switch_players_and_start_game for right/B).

Because Karambol reflexes `start_game` / `switch_players_and_start_game` don't accept parameters (they only read `element.dataset[:id]`), we augment them with an optional `element.dataset[:bk2_first_set_mode]` read: if present, write into bk2_options BEFORE calling the Karambol state-transition.

Output: one view file branched + one reflex-action augmentation.
</objective>
