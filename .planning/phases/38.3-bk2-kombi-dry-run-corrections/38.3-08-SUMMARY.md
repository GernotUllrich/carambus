# Phase 38.3 Plan 08: BK2 Shootout→Playing State Initialization (I6 Gap Closure) Summary

**Public `Bk2Kombi::AdvanceMatchState.initialize_bk2_state!` entry-point wired into all 3 shootout reflex paths, ensuring bk2_state is fully populated before AASM transitions to playing — closes I6 blocker that caused the GAP-05 "Spiel nicht initialisiert" fallback banner**

## Accomplishments

- Exposed the existing private `init_state_if_missing!` as a public class method `Bk2Kombi::AdvanceMatchState.initialize_bk2_state!(table_monitor)` — delegates via `send`, preserving private surface while enabling the reflex to call it
- Wired the init call into all 3 reflex paths that reach `finish_shootout!`: `start_game`, `switch_players_and_start_game` (button paths — after Plan 38.3-05 bk2_options write), and `key_d` (keyboard path — relies on DEFAULT_FIRST_SET_MODE fallback when no mode set)
- Added 4 unit tests (T1..T4) confirming DZ-mode init, SP-mode init with non-default options, idempotency on already-populated state, and default-mode fallback for the key_d path; added 2 system tests (T14..T15) exercising the full shootout→playing model transition sequence

## Root Cause Confirmation (Candidate 1)

`init_state_if_missing!` was private and only reached via `Bk2Kombi::AdvanceMatchState.call(...)` — which requires a `shot_payload` and is only invoked from scoring paths (dormant in Variante B). The Plan 38.3-05 shootout→playing path wrote `bk2_options[first_set_mode]` correctly but never triggered state initialization. Result: `bk2_state_uninitialized?` returned `true` after the shootout click, and `_show.html.erb:101` rendered the GAP-05 fallback banner instead of the playing scoreboard layout.

## Decisions Made

- **`send(:init_state_if_missing!)` instead of making it public:** Keeps the private method private (no new public surface on the instance); the wrapper method owns the `save!` call contract. Same delegation pattern used by `Bk2Kombi::CommitInning` (Plan 38.3-01).
- **Three call sites required:** The plan-checker (W1) identified `key_d` as a third path reaching `finish_shootout!`. All 3 are now covered — leaving `key_d` uncovered would have partially re-introduced I6 on keyboard-triggered shootout confirmations.
- **System tests inline model transitions:** Rather than using Capybara click (which requires StimulusReflex ActionCable round-trip in test mode — flaky), T14/T15 directly exercise the model methods in the same order as the reflex. This is the established pattern from T4..T13.
