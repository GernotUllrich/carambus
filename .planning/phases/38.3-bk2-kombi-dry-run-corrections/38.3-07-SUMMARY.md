## Decisions Made

**Fixture choice — `table_monitors(:one)` not `table_monitors(:free)`:**
The `TableMonitorsController#show` action redirects to the location tables screen when `game_id` is blank. The `:free` fixture has no associated `Table` record (no `Table` row with `table_monitor_id: 50_000_003`). The `:one` fixture has `id: 50_000_001` which matches `tables(:one).table_monitor_id`, so `@tm.table` resolves correctly. Using `:free` caused a 500 error in Capybara visits.

**ERB source assertion for T1/T2/T13:**
Alpine.js-driven content (hidden inputs with `:value` bindings, x-data state slots) is only rendered in the DOM after JS hydration. Asserting the ERB template source file directly is deterministic, fast, and immune to JS timing. Applied to all three template-level guards.

**Direct CommitInning.call for T8-T11:**
StimulusReflex round-trips require an ActionCable server backed by Redis. The headless Capybara session does not start ActionCable in-process. Calling `Bk2Kombi::CommitInning.call` directly exercises the same business logic path that the reflex invokes, and the service layer tests already cover the reflex wiring (38.3-04). This avoids flaky `sleep N` patterns.
