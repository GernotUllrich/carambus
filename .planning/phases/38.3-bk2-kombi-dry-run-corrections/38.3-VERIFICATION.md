### 1. Full Reflex Round-Trip: accumulate + commit via swap icon

**Test:** Start a BK2 match in playing state. Click +5 then +10 on the active player. Observe the small red inning-score slot. Click the swap icon.
**Expected:** Inning-score shows 15 before commit. After swap, set_scores.1.playera increases by 15 (SP additive) or opponent gets +15 (DZ), player_at_table flips, inning-score resets to 0.
**Why human:** The full `add_n` -> `innings_redo_list[-1]` -> `next_step` -> `bk2_kombi_commit_if_active` -> `CommitInning` reflex chain requires an ActionCable-backed browser session. System tests bypass this chain by calling `Bk2Kombi::CommitInning.call` directly.

### 2. Start-game flow: detail-view config -> warmup -> shootout -> playing scoreboard

**Test:** Open the BK2 detail-view. Select BK2 50. Set DZ-max to 3, SP-max to 7. Click Start Game. Go through warmup, arrive at shootout. Confirm 4 buttons visible. Click "Spieler A — Serienspiel". Confirm the scoreboard phase chip shows "Serienspiel" and the remaining badge shows "Aufnahmen ubrig: 7".
**Expected:** Complete flow; `bk2_options["first_set_mode"] == "serienspiel"`, `bk2_state["innings_left_in_set"] == 7` after init.
**Why human:** Multi-step AASM transition (warmup -> match_shootout -> playing) is only testable with a live browser session. Alpine.js-driven hidden inputs only bind after JS hydration.

### 3. Visual layout D-20 compliance at 1024x768 landscape

**Test:** Open the BK2 scoreboard in fullscreen on a 1024x768 or similar tablet-resolution viewport. Verify the phase chip + remaining badge fit within the center strip without overflow. Verify player cards show no GD/HS rows and the Ziel slot shows `set_target_points`.
**Expected:** No horizontal or vertical scroll. Phase chip visible. GD/HS rows absent. Ziel shows "50" (or configured value).
**Why human:** Layout overflow can only be verified by visual inspection at the target resolution. Capybara headless tests run at default viewport and don't catch CSS overflow issues.
