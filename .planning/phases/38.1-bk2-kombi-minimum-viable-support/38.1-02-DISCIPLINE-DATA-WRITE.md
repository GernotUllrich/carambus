## Background

Discipline id 107 (`BK2-Kombi`, `table_kind = "Small Billard"`) is a global record
(id < 50,000,000). Its `data` column (text, JSON-encoded) must contain
`{ "free_game_form" => "bk2_kombi" }` so that `GameSetup.derive_free_game_form`
picks it up via the authoritative path (Priority 1) rather than the name-match
fallback (Priority 2, which logs a reconciliation warning).

**The name-match fallback WILL work** on tournament day even without this write —
`discipline.name == "BK2-Kombi"` is the safety net. But completing this write
makes the derivation authoritative, removes the warning from the logs, and
satisfies T-38.1-05 (Tamper mitigation).

## Path B (FALLBACK — local unprotected write from carambus_bcw)

Use ONLY if sync timing is too tight before 2026-05-02 and the name-match fallback
is considered insufficient (it is functional but logs a reconciliation warning).

This path explicitly bypasses `LocalProtector` on a global record. It is an
audited, one-shot bypass — the `unprotected=true` flag is instance-only, not
persisted. See T-38.1-07 (Elevation of Privilege) in the Plan 02 threat model.

**Requires:** Explicit human authorization (scenario-management SKILL.md — not
executed by Claude in Normal Mode).

### Commands (run in carambus_bcw Rails console or runner)

```ruby
# Step 1: verify current state
puts Discipline.find(107).name
puts Discipline.find(107).data.inspect

# Step 2: write with LocalProtector bypass
d = Discipline.find(107)
d.unprotected = true
d.data = (d.data || {}).merge("free_game_form" => "bk2_kombi")
d.save!
puts "BCW Discipline 107 data now: #{d.reload.data.inspect}"
# Expected: BCW Discipline 107 data now: {"free_game_form"=>"bk2_kombi"}

# Step 3: verify
puts Discipline.find(107).data["free_game_form"].inspect
# Expected: "bk2_kombi"
```

Or as a one-liner:

```bash
bin/rails runner 'd = Discipline.find(107); d.unprotected = true; d.data = (d.data || {}).merge("free_game_form" => "bk2_kombi"); d.save!; puts "BCW Discipline 107 data now: #{d.reload.data.inspect}"'
```
