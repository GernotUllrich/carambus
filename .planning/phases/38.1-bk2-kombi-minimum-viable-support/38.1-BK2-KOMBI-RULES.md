## Disciplin-Einordnung

BK2-Kombi combines two playing phases:

- **Direkter Zweikampf** — two shots per turn; bonus shot when "5" (all 5 pins) or "6" (all 5 pins + carom) is achieved
- **Serienspiel** — player continues until a foul or zero-point shot

Authoritative rules:
- https://billardregel.de/kegel/billardkegel-dbu/ (BK2-Kombi format)
- https://billardregel.de/kegel/billardkegel-dbu/04-die-fehler/ (fouls §4)
- https://www.bcdorff.de/Sport/Spielregeln-Kegeln.pdf (DBU Billard-Kegeln rules)
- https://www.bk-portal.de/wp-content/uploads/2014/10/Ordnung-Spielregeln.pdf (BKV Ordnung)
- https://sachsen-billard.de/kegel/dmm-bk2-kombi (DMM BK2-Kombi)

## Rechenlogik (3 steps per shot)

1. **Rohpunkte** — pin hits, middle-pin special case, carom, passage
2. **Fehlerprüfung** — if foul, entire shot result goes negative/to opponent
3. **Phasenlogik** — bonus shot (Direkter Zweikampf) or series continuation (Serienspiel)

### Base scoring values

- Every fallen pin: **1 pt**
- Middle pin alone from full pin image: **2 pts**
- Carom: **1 pt**
- Passage: **1 pt** (config-dependent per rule variant)

### Foul logic

> Fouls are NOT a boolean — they're concrete negative evaluations. The
> entire shot result turns negative and is credited to the opponent.

Published foul values (DBU rules):

| Foul code | Value | Rule |
|-----------|-------|------|
| `wrong_ball` | -6 | false ball played |
| `no_object_ball_hit` (with band) | -1 | miss, ball hit band |
| `no_object_ball_hit` (no band) | -2 | clean miss |
| `premature_shot` | -1 | shot too early |
| `direct_play_on_pins` | max(1, fallen_pins) | direct pin play |
| (other) | -1 | default fallback |

When a foul occurs: `total_to_opponent = raw[:total] + foul[:foul_points]`.

## Rails-Service-Schnitt (Perplexity's proposal)

```ruby
class Bk2Kombi::ScoreShot
  def initialize(match:, shot_payload:)
    @match = match
    @rules = match.rule_set.config.deep_symbolize_keys
    @state = match.state.deep_symbolize_keys
    @shot  = shot_payload.deep_symbolize_keys
  end

  def call
    raw = calculate_raw_points
    foul = calculate_foul_points
    scoring = foul[:foul] ? apply_foul_scoring(raw, foul) : apply_positive_scoring(raw)
    transitions = calculate_transitions(scoring)
    {
      scoring: scoring,
      transitions: transitions
    }
  end
end
```

### Rohpunkte berechnen

```ruby
def calculate_raw_points
  obs = @shot.fetch(:observations)

  pin_points =
    if obs[:middle_pin_only] && @shot.dig(:table_snapshot, :full_pin_image)
      @rules.dig(:scoring, :middle_pin_only_from_full_image).to_i
    else
      obs[:fallen_pins].to_i * @rules.dig(:scoring, :pin).to_i
    end

  carom_points = 0
  carom_points += @rules.dig(:scoring, :true_carom).to_i if obs[:true_carom]
  carom_points += @rules.dig(:scoring, :false_carom_with_pin_or_passage).to_i if obs[:false_carom]

  passage_points = obs[:passages].to_i * @rules.dig(:scoring, :passage).to_i

  total = pin_points + carom_points + passage_points

  {
    pin_points: pin_points,
    carom_points: carom_points,
    passage_points: passage_points,
    total: total
  }
end
```

### Fehler berechnen

```ruby
def calculate_foul_points
  obs = @shot.fetch(:observations)
  foul_code = obs[:foul_code]&.to_sym

  return { foul: false, foul_points: 0 } unless obs[:foul]

  foul_points =
    case foul_code
    when :wrong_ball
      6
    when :no_object_ball_hit
      obs[:band_hit] ? 1 : 2
    when :premature_shot
      1
    when :direct_play_on_pins
      [1, obs[:fallen_pins].to_i].max
    else
      1
    end

  { foul: true, foul_code: foul_code, foul_points: foul_points }
end
```

### Punkte zuweisen

```ruby
def apply_positive_scoring(raw)
  {
    foul: false,
    points_for_current_player: raw[:total],
    points_for_opponent: 0,
    raw_total: raw[:total],
    credited_total: raw[:total]
  }
end

def apply_foul_scoring(raw, foul)
  total_to_opponent = raw[:total] + foul[:foul_points]
  {
    foul: true,
    foul_code: foul[:foul_code],
    foul_points: foul[:foul_points],
    points_for_current_player: 0,
    points_for_opponent: total_to_opponent,
    raw_total: raw[:total],
    credited_total: total_to_opponent
  }
end
```

### Match-State updaten

BK2-Kombi sets run to configurable target 50 / 60 / 70 points per Ausschreibung.

```ruby
def apply_score_to_state!(scoring)
  set_no = @state[:current_set_number].to_s
  current = @state[:player_at_table].to_sym
  opponent = current == :player_a ? :player_b : :player_a

  @state[:set_scores][set_no][current.to_s] += scoring[:points_for_current_player]
  @state[:set_scores][set_no][opponent.to_s] += scoring[:points_for_opponent]
end
```

### Bonusstoß + Spielerwechsel

```ruby
def calculate_transitions(scoring)
  phase = @state[:current_phase].to_sym
  obs = @shot.fetch(:observations)

  if phase == :direkter_zweikampf
    bonus = !scoring[:foul] && bonus_shot?(obs)

    if bonus
      {
        bonus_shot_awarded: true,
        turn_ends: false,
        next_player_at_table: @state[:player_at_table],
        next_shots_left_in_turn: [@state[:shots_left_in_turn].to_i, 1].max
      }
    else
      next_shots = @state[:shots_left_in_turn].to_i - 1
      turn_ends = scoring[:foul] || next_shots <= 0
      {
        bonus_shot_awarded: false,
        turn_ends: turn_ends,
        next_player_at_table: turn_ends ? opponent_position(@state[:player_at_table]) : @state[:player_at_table],
        next_shots_left_in_turn: turn_ends ? 2 : next_shots
      }
    end
  else
    # Serienspiel
    turn_ends = scoring[:foul] || scoring[:points_for_current_player].zero?
    {
      bonus_shot_awarded: false,
      turn_ends: turn_ends,
      next_player_at_table: turn_ends ? opponent_position(@state[:player_at_table]) : @state[:player_at_table],
      next_shots_left_in_turn: nil
    }
  end
end

def bonus_shot?(obs)
  all_pins = obs[:fallen_pins].to_i == 5
  all_pins_and_carom = all_pins && (obs[:true_carom] || obs[:false_carom])
  all_pins || all_pins_and_carom
end
```

### Satzende + Matchende

```ruby
def close_set_if_needed!
  set_no = @state[:current_set_number].to_s
  target = @match.set_target_points

  a = @state[:set_scores][set_no]["player_a"]
  b = @state[:set_scores][set_no]["player_b"]

  return unless a >= target || b >= target

  winner = a >= target ? "player_a" : "player_b"
  @state[:sets_won][winner] += 1
  @state[:set_finished] = true
  @state[:set_winner] = winner
end
```

At set-split (1-1), a third deciding set is played. Match winner = first to 2 sets.

## Adaption an Carambus

The Perplexity transcript assumes a greenfield Rails API. Phase 38.1 adapts
to the existing Carambus MVC structure:

- **No new `Match` / `RuleSet` ActiveRecord models.** State lives in
  `TableMonitor.data["bk2_state"]` hash (same pattern snooker uses with
  `data["snooker_state"]`). Rule config lives in `discipline.data` or a
  module-level constant for Phase 38.1; a config-table migration is deferred.
- **`Bk2Kombi::ScoreShot` → `app/services/bk2_kombi/score_shot.rb`**
  (singular service, pattern matches existing `app/services/table_monitor/*`).
- **`Bk2Kombi::AdvanceMatchState` → `app/services/bk2_kombi/advance_match_state.rb`**.
- **Invocation:** `TableMonitor::ResultRecorder` (or a reflex) checks
  `data["free_game_form"] == "bk2_kombi"` and dispatches to `Bk2Kombi::ScoreShot`
  instead of `score_engine.add_n_balls`.
- **innings_list shape:** BK2-Kombi shots are finer-grained than karambol
  innings (a turn is N shots, not one entry). The `innings_list` per player
  stores per-turn credited points for compatibility with the existing
  history/protocol UI; per-shot detail lives in a new `shots_list`
  alongside it. Planner decides exact shape.

## Zweckmäßige Service-Aufteilung

- `Bk2Kombi::ScoreShot` — evaluates one shot, returns scoring + proposed transitions (pure / no side effects)
- `Bk2Kombi::AdvanceMatchState` — applies scoring + transitions to TableMonitor.data, closes set/match when target hit

This separation makes the point math independently testable from the
match-flow transitions.
