# Phase 38.1 Plan 03: BK2-Kombi Scoring Services Summary

**One-liner:** BK2-Kombi shot scoring implemented as pure `ScoreShot` function (DEFAULT_RULES constant, D-15/D-16 exact values) + `AdvanceMatchState` state applier with set/match close, idempotency guard, and 40 passing tests.

## What Was Built

### Bk2Kombi::ScoreShot (217 lines)

Pure shot evaluator — no DB access, no side effects. Returns `{scoring:, transitions:}`.

**DEFAULT_RULES constant (frozen):**
```ruby
DEFAULT_RULES = {
  scoring: {
    pin: 1,
    middle_pin_only_from_full_image: 2,
    true_carom: 1,
    false_carom_with_pin_or_passage: 1,
    passage: 1
  },
  foul_values: {
    wrong_ball: -6,
    no_object_ball_hit_with_band: -1,
    no_object_ball_hit_no_band: -2,
    premature_shot: -1,
    direct_play_on_pins: nil, # computed: -[1, fallen_pins].max
    other: -1
  }
}.freeze
```

**B3 fix (direct_play_on_pins sign):**
```ruby
when :direct_play_on_pins
  -[1, obs[:fallen_pins].to_i].max
  # fallen_pins=0 → -1 (floor), fallen_pins=3 → -3 (always NEGATIVE)
```

**Foul credit convention:** `points_for_opponent = |foul_points|` only (not `raw + penalty`). Verified by test 10: `direct_play_on_pins, fallen_pins=3` → `points_for_opponent=3`, not 6.

### Bk2Kombi::AdvanceMatchState (151 lines)

Applies ScoreShot output to `TableMonitor.data['bk2_state']`, persists via `tm.save!`.

| Method | Purpose |
|--------|---------|
| `init_state_if_missing!` | Creates fresh bk2_state if nil |
| `apply_scoring!` | Writes points to set_scores[current_set][player] |
| `apply_transitions!` | Updates player_at_table + shots_left_in_turn |
| `close_set_if_reached!` | Increments sets_won, advances set number, resets phase, alternates kickoff |
| `close_match_if_reached!` | Sets match_finished=true + match_winner at first-to-2 |
| `already_applied?` / `record_sequence!` | Idempotency guard (T-38.1-13) |

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] apply_foul_scoring used raw + penalty instead of penalty only**

- **Found during:** Task 1 GREEN — test 10 failed: `direct_play_on_pins, fallen_pins=3` expected `points_for_opponent=3` but got 6
- **Issue:** Plan interface spec said `points_for_opponent = |raw_total| + |foul_points|`. For `direct_play_on_pins`, the raw total includes the fallen pins (3), and the penalty is also 3 (max(1,3)). That double-counts.
- **Fix:** Changed `apply_foul_scoring` to use `opponent_credit = foul_info[:foul_points].abs` only. The foul penalty already encodes the fallen-pin consequence; raw points don't separately transfer on a foul.
- **Verification:** Test 10 (fallen_pins=3 → opp=3) and test 10b (fallen_pins=0 → opp=1) both pass. Tests 6-9, 11-12 also pass (all have fallen_pins=0, so raw=0 either way).
- **Files modified:** `app/services/bk2_kombi/score_shot.rb`
- **Commit:** 2320f281

**2. [Rule 1 - Bug] Integration test 3 preset wrong player for set 2 close**

- **Found during:** Task 3 — test failed: "playera must have 2 sets won. Expected: 2 Actual: 1"
- **Issue:** After set 1 closes, kickoff alternates: `player_at_table` becomes "playerb". The test preseted `set_scores["2"]["playera"] = 47` and fired a 5-pin shot, but the shot applied to "playerb" (the actual player at table).
- **Fix:** Test now explicitly resets `player_at_table` to "playera" before the set 2 close shot, simulating a legitimate playera turn.
- **Files modified:** `test/integration/bk2_kombi_dispatch_integration_test.rb`
- **Commit:** 06679a30
