### `TableMonitorReflex#bk2_kombi_submit_shot`

Added at end of `table_monitor_reflex.rb` (after `balls_goal_b`):

- Discipline gate: `return unless @table_monitor.data["free_game_form"] == "bk2_kombi"` — mirrors snooker action pattern exactly
- Builds D-13 payload from `element.dataset`: coerces `.to_i` for integers, `== "true"` for booleans, `.presence&.to_sym` for foul_code
- **B2 FIX**: reads `element.dataset["full_pin_image"]` which the Stimulus controller populates from `fullPinImageTarget.checked`
- Calls `Bk2Kombi::AdvanceMatchState.call(table_monitor:, shot_payload:)`
- `morph :nothing` — avoids global CableReady broadcast; scoped broadcast happens via `CableReady::Updatable` callbacks when `tm.save!` fires inside AdvanceMatchState

**Broadcast idiom adopted:** `morph :nothing` + `CableReady::Updatable` on `tm.save!`. This is the same pattern used by the snooker actions at lines 439, 462, 472, 485, 499, 513 of `table_monitor_reflex.rb` — all end with `@table_monitor.save` and rely on the model's broadcast callbacks. No explicit `cable_ready[...].broadcast_to(...)` call needed; the `save!` in AdvanceMatchState triggers it.

### Architecture Decision: Shot-submit test approach

- **Plan said:** System test with Capybara click on the submit button triggers the StimulusReflex
- **Reality:** No existing system tests exercise `data-reflex` clicks directly — the pattern in this codebase is to trigger state changes via Ruby and assert DOM updates. StimulusReflex click paths in Capybara require a full JS runtime with WebSocket + StimulusReflex initialization that is fragile.
- **Resolution:** DOM rendering assertions via Capybara (tests 1–2); shot-submit assertions via direct `Bk2Kombi::AdvanceMatchState.call` (tests 3–5). Plan explicitly permits this fallback: "fall back to a controller test... but the shot-submit tests MUST remain integration-level."
- **Impact:** B2 coverage is fully satisfied at integration level — the complete Ruby dispatch chain is exercised.

### Stimulus controller registration

- **Plan said:** Manually add `import Bk2KombiShotController` to `index.js`
- **Reality:** `index.js` uses `import controllers from "./**/*_controller.js"` which auto-discovers all `*_controller.js` files. No manual edit needed. The controller name `bk2-kombi-shot` is derived from the filename automatically.
