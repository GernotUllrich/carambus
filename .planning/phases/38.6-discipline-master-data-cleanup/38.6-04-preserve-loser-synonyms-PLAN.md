# Plan 38.6-04 — Preserve loser synonyms during merge

<objective>
Amend `script/merge_bk_disciplines.rb` so that **before** `loser.destroy` runs, the loser's `name` and its existing `synonyms` text-column entries are appended to the winner's `synonyms` field. The `Discipline` model already has a `before_save :update_synonyms` callback (`app/models/discipline.rb:43`) that joins newline-separated entries and adds `self.name` automatically — extend that pattern; do not bypass it. ## Task 1 — Amend `merge_loser_into_winner` to preserve loser aliases

**Edit target:** `/Volumes/EXT2TB/gullrich/DEV/carambus/carambus_api/script/merge_bk_disciplines.rb`

**Location:** Inside `merge_loser_into_winner(loser, winner)` (around line 254), insert the synonym-preservation block **immediately before** the existing `# --- Finally: destroy the loser ---` block (around line 325-331). The block must run inside the same `Discipline.transaction` so a rollback also undoes the synonyms change in dry-run mode.

**Pattern to add:**

```ruby
# --- Preserve loser aliases in winner.synonyms ---
# Capture loser.name + loser.synonyms (newline-separated) so scrapers and
# legacy lookups still resolve via Discipline#synonyms after destroy.
loser_aliases = ([loser.name] + loser.synonyms.to_s.split("\n"))
                  .map { |s| s.to_s.strip }
                  .reject(&:empty?)
existing       = winner.synonyms.to_s.split("\n").map(&:strip).reject(&:empty?)
merged         = (existing + loser_aliases).uniq
if merged.sort != existing.sort
  winner.synonyms = merged.join("\n")
  winner.save!  # before_save :update_synonyms keeps dedup + ensures name present
  protocol_append("- synonyms preserved on winner #{winner.id}: added #{(merged - existing).inspect}")
else
  protocol_append("- synonyms unchanged on winner #{winner.id} (loser aliases already present)")
end
```

- [ ] `winner.save!` is used (not `update_columns`) so PaperTrail emits a Version row on winner
- [ ] The transaction wrapper around the merge already covers the new code — no new transaction
- [ ] `protocol_append` lines added so the run-time Markdown protocol reports the change

**Commit message:** `feat(38.6-04): preserve loser name + synonyms in winner.synonyms before destroy`
