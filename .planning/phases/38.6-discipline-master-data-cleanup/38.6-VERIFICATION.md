## Known Warnings (from 38.6-REVIEW.md) — Advisory Only

None of the 4 warnings from the code review block goal achievement. They are documented here for production-deployment awareness:

**WR-01 — BK_NAME_TO_FORM stale key (action required before production)**

`app/services/table_monitor/game_setup.rb:243` still maps `"BK2-Kombi" => "bk2_kombi"`. After the merge script renames winner-107 to "BK-2kombi", the `derive_free_game_form` priority-2 name-based fallback and `BkParamResolver.lookup_discipline("bk2_kombi")` both silently fail to match any DB row. Runtime impact is mitigated by priority-1 `data["free_game_form"]` firing first for all properly-seeded rows, and BK-2kombi resolving via component disciplines rather than direct lookup. However, the entry is a silent lie post-merge and should be fixed in a follow-up plan before the production merge runs. The code review suggests keeping both keys as aliases until all local servers have synced.

**WR-02 — Dead rescue ActiveRecord::Rollback (style, no data risk)**

`merge_loser_into_winner` has a `rescue ActiveRecord::Rollback` at method scope outside the transaction block. Since `ActiveRecord::Rollback` is swallowed by the transaction and never re-raises, this rescue never fires. Cosmetic fix for a future maintenance pass.

**WR-03 — exit 0 bypasses dry-run rollback verification**

`pre_flight!` calls `exit 0` when all losers are absent. When wrapped in the dry-run transaction block, `exit` unwinds without triggering `raise ActiveRecord::Rollback`, so the post-rollback verification block is never reached. No data risk (DB was already clean). Fix for future dry-run reliability.

**WR-04 — Two seed-replay tests still use "BK2-Kombi" (intentional carve-out)**

`discipline_test.rb` lines 201, 208, 231 (T-P5 seed-replay tests) deliberately retain "BK2-Kombi" to stay synchronized with `script/seed_bk2_disciplines.rb` which is out of scope for Phase 38.6. The carve-out is documented in Plan 02 SUMMARY. These tests pass correctly; they characterize seed behavior, not the canonical DB name.
