## Seed-Replay Carve-Out (Deliberately Preserved)

Three occurrences of `"BK2-Kombi"` remain in `test/models/discipline_test.rb` at lines 201, 208, and 231. These are inside the nachstoss seed-replay tests (`T-P5-seed-only-bk2kombi-has-flag` and `T-P5-bk2kombi-keeps-nachstoss`) which programmatically replay `script/seed_bk2_disciplines.rb`'s `find_or_initialize_by(name: "BK2-Kombi")` block.

The seed file (`script/seed_bk2_disciplines.rb`) is **out-of-scope for Phase 38.6** — it runs in production where Plan 01's merge script handles the rename via `Discipline.find(107).update!(name: "BK-2kombi")`. The seed uses `find_or_initialize_by(name: "BK2-Kombi")` for the non-107 disciplines and `find(107)` as an id-anchor for the BK-2kombi backfill. Id-anchor lookups continue to work post-rename. Updating the seed's `find_or_initialize_by` lookup to `"BK-2kombi"` is a follow-up task for a future phase once the production merge script has run successfully.

Until the seed is updated, these seed-replay tests must keep `"BK2-Kombi"` strings to remain faithful characterizations of the seed's behavior. The 3 kept occurrences are the only remaining `"BK2-Kombi"` strings in `test/models/discipline_test.rb`.
