### Q1.2: Detail clarifications (user-driven prose response, no AskUserQuestion)

User corrected several assumptions from the initial ASCII mockup:

- **Phase switches NOT mid-set.** BK2 is a multi-set match like Karambol
  best-of-3; each set is one mode (DZ or SP), mode sequence chosen by the
  Anstossender at match start. Satz 3 only at 1:1, same mode as Satz 1.
- **Match score (sets won) already has a Karambol slot** next to the names
  in the header — reuse it for D-06.
- **Shots-übrig NOT only in DZ** — belongs in SP too (later corrected to:
  DZ has shots-per-Aufnahme=2+bonus; SP has Aufnahmen-per-Set cap ~5,
  "ausschreibungsabhängig").
- **Set history (S1 S2 S3 per-player) unnecessary** — only who-won-the-set
  matters in the UI; per-Aufnahme detail goes in the Protokoll.
- **Current-set score is live** and may go transiently negative during an
  Aufnahme; rollback on negative Aufnahmewechsel → opponent credited.
- **GD/HS can be removed.** Ziel = current set target.
- **Inning counter can be removed.** Disciplin/Spiel/Tisch names stay.
- **Bottom input bar may grow at the expense of the player cards.**

## Rule Clarification (mid-discussion, before Area 2)

User supplied the BK2 rulebook excerpt clarifying shot/inning limits:

- **DZ:** 2 Stöße pro Aufnahme, Bonusstoss bei "5" (alle Kegel) oder "6"
  (alle Kegel + Karambolage).
- **SP:** Spieler bleibt am Tisch solange er punktet; in Variante
  "70/5 Aufnahmen" max 5 Aufnahmen pro Spieler ("Ausschreibung maßgeblich").

User flagged that rule interpretation must be confirmed with officials.

**Design implication:** Badge is phase-sensitive (Stöße übrig in DZ,
Aufnahmen übrig in SP) AND limits are configurable via `bk2_options`
(not hard-coded) — post-clarification rule changes become config changes.
