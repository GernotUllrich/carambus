# Phase 38.2: BK2-Kombi scoreboard UX re-alignment - Context

**Gathered:** 2026-04-23
**Status:** Ready for planning

<domain>
## Phase Boundary

Replace the Phase 38.1 Plan 04 BK2-Kombi scoreboard UI (`_show_bk2_kombi.html.erb`,
`_scoreboard_bk2_kombi.html.erb`, `bk2_kombi_shot_controller.js`,
`test/system/bk2_kombi_scoreboard_test.rb`) with a variant that structurally
preserves the existing Karambol scoreboard layout so volunteer scorers familiar
with the Karambol UI are not disoriented on tournament day (2026-05-02).

In scope:
- New `_show_bk2_kombi.html.erb` + `_scoreboard_bk2_kombi.html.erb` as 1:1
  structural copies of the Karambol equivalents, with BK2-specific deltas added
  additively.
- New shot-entry bottom-bar that replaces the Karambol `-10/-5/-1 | +1/+5/+10`
  button row.
- Start-game form: replace the single "Spieler A stösst an" button with two
  buttons per player — one per first-set mode (DZ / Serienspiel).
- Resolution of UAT-GAP-02..GAP-05 from Phase 38.1 dry-run feedback (inline,
  because the affected files are being rewritten anyway).
- Service-layer adaptation where strictly required for the new Badge (phase-
  sensitive `shots_left_in_turn` vs `innings_left_in_set`).
- Configurable shot/inning limits in `bk2_options` (not hard-coded) so post-
  phase rule clarifications land as config changes, not code.

Out of scope:
- Changes to `Bk2Kombi::ScoreShot` or `Bk2Kombi::AdvanceMatchState` scoring
  math (DEFAULT_RULES, D-15/D-16 literals) — only augmentation if the
  phase/inning accounting requires new state fields.
- Changes to `TableMonitorsController#start_game` beyond accepting the new
  `first_set_mode` parameter.
- Changes to `Discipline::BK2_DISCIPLINE_MAP`, `carambus.yml` BK2 quick
  preset, or the `/locations/:md5?sb_state=free_game` quick-button flow — all
  stay as shipped in Phase 38.1 Plan 06.
- Plan 38.1-05 (dry-run UAT + fallback drill) — remains deferred until this
  phase ships.

<decisions>
## Implementation Decisions

### Overall Redesign Strategy

- **D-01:** Variante B — create a **new BK2 partial that is a structural 1:1
  copy of the Karambol scoreboard** (`_show.html.erb` +
  `_scoreboard_free_game_karambol.html.erb` if extracted; otherwise the
  scoreboard section inline in `_show.html.erb`). BK2-specific deltas are
  **additive only**. Two parallel templates (Karambol + BK2) rather than a
  single template with conditional branches.
- **D-02:** Services (`Bk2Kombi::ScoreShot`, `Bk2Kombi::AdvanceMatchState`),
  reflex (`TableMonitorReflex#bk2_kombi_submit_shot`),
  `Discipline::BK2_DISCIPLINE_MAP`, `carambus.yml` BK2 preset, and
  `TableMonitorsController#start_game` BK2 branch **stay as shipped in
  38.1 Plan 06**. Only augmentations allowed, not rewrites.

### Scoreboard Layout — Slot Mapping

- **D-03:** Minimal-invasive re-purposing of existing Karambol slots. BK2
  deltas inhabit the same visual positions as their Karambol counterparts —
  new positions only where no Karambol equivalent exists.
- **D-04:** **Center area (Karambol inning-counter slot):** the
  Karambol inning number ("1") and "of 20" text are replaced by the current
  **set number and phase chip** (e.g., "Satz 2 / 3" on top, "Serienspiel" or
  "Direkter Zweikampf" underneath as a colored chip). The discipline name
  ("BK2-Kombi") and table name ("Tisch 7") stay visible in the Karambol
  position — important for tournament context.
- **D-05:** **Player card "Ziel" slot:** `Ziel: N` remains at the Karambol
  position and shows the **current set's target** (50/60/70). Karambol GD
  (general Durchschnitt) and HS (höchste Serie) rows are **removed** — they
  do not apply to BK2 scoring.
- **D-06:** **Match score (sets won, 0:0 → 1:0 → 2:0 / 2:1):** uses the
  **existing Karambol multi-set slot next to the player names in the top
  header** — Karambol already has this slot for best-of-N formats. Zero
  new layout real estate needed; parity is maintained.
- **D-07:** **Current-set score (large number per player):**
  - Shows **only the current set's point total**, not cumulative.
  - Updated **live during an in-progress Aufnahme** (inning) as the scorer
    enters successive shots. The displayed number may go **transiently
    negative**.
  - On **Aufnahmewechsel with negative inning score:** the player's current-
    set score rolls back to the pre-Aufnahme value, and the absolute value
    of the negative amount is added to the opponent's current-set score
    (credited as positive points).
- **D-08:** **Inning counter:** removed from the BK2 scoreboard. Not
  applicable to BK2 scoring model.
- **D-09:** **Discipline name / game name / table name:** stay visible in
  Karambol positions. Tournament context (name, Spielname, Tisch) is
  particularly important on BCW club tables.
- **D-10:** **Home / Cancel / Continue navigation modal:** unchanged from
  Karambol — copied verbatim from `_show.html.erb:72–95`. This closes
  UAT-GAP-04 automatically.
- **D-11:** **Phase-sensitive "remaining" badge in the center area:**
  - DZ set: labeled **"Stöße übrig: N"** — N counts down per-shot within
    the current Aufnahme. Starts at the configured limit (default 2).
    Bonus shot ("5" or "6" result) increments by 1 temporarily. Resets to
    the configured limit on Aufnahmewechsel.
  - Serienspiel set: labeled **"Aufnahmen übrig: M"** — M counts down per-
    Aufnahme on the set level. Starts at the configured limit (default 5
    based on "70/5 Aufnahmen" variant; competition-specific).
- **D-12:** **Rule limits are configurable, not hard-coded.** Values for
  DZ max-shots-per-Aufnahme and Serienspiel max-Aufnahmen-per-Set live in
  `bk2_options` (and/or `discipline.data`) and are read by the service
  layer. User noted rule interpretation must be confirmed with officials
  ("muss klären mit Spezialisten") — making these config-driven ensures
  post-clarification rule tweaks are config changes, not code changes.
- **D-13:** **Set history (S1 S2 S3 per-player breakdown):** **removed**
  from the scoreboard. Only the set-win counter in the match-score slot
  (D-06) shows match progress. Individual Aufnahmen per set are recorded
  in the **Protokoll** (PDF export), not in the live scoreboard UI.

### Start-Game UI — Anstoss Mode Selector

- **D-14:** **Replace the single "Spieler A stösst an" button with two
  buttons per player**, encoding both Anstossender and first-set mode in
  one click:
  - `Spieler A stösst an — Direkter Zweikampf` → Satz 1 = DZ, Satz 2 = SP
  - `Spieler A stösst an — Serienspiel` → Satz 1 = SP, Satz 2 = DZ
  - Analog for Spieler B (4 buttons total)
  - Satz 3 (only played at 1:1) uses the **same mode as Satz 1**
  - Implementation location: Karambol detail view
    (`app/views/locations/scoreboard_free_game_karambol_new.html.erb`) BK2
    branch. Payload: new `bk2_options[:first_set_mode]` field carrying
    `"direkter_zweikampf"` or `"serienspiel"`. Services use this to derive
    Satz 1/2/3 modes.

### Shot-Entry Bottom Bar

- **D-15:** **Mehrzeilige Bottom-Bar** — the Karambol
  `-10/-5/-1 | reset | +1/+5/+10` button row is **replaced** by a 2–3-line
  BK2 shot-entry form occupying the full width of the screen. Player cards
  remain at Karambol size — layout symmetry with the familiar scoreboard
  is preserved.
- **D-16:** **Bottom-bar content layout:**
  - **Row 1 (amber accent):** `[ ] Volles Kegelbild vor Stoss` checkbox
    with short hint text. Must be checked **before** entering fallen pins
    to activate the D-15 "middle-pin-only from full image = 2 points"
    rule.
  - **Row 2 (main inputs):** `Gefallene Kegel: [0..5 ▾]` dropdown,
    `[ ] Mittelpin` checkbox, `[ ] Echter Karambol.`,
    `[ ] Unechter Karambol.`
  - **Row 3 (secondary + submit):** `Durchläufe: [N]` number input,
    `[ ] Fehler` checkbox → reveals `Fehlercode: [… ▾]` dropdown and
    (conditional) `[ ] Bande getroffen` flag, followed by the
    `[ STOSS ERFASSEN ]` submit button on the right.

### Plan 04 Artifact Handling

- **D-17:** **Inplace overwrite** of all four Plan 04 artifacts at their
  existing paths:
  - `app/views/table_monitors/_show_bk2_kombi.html.erb` (39 Z → new,
    1:1 from `_show.html.erb` with BK2 dispatcher adjustments)
  - `app/views/table_monitors/_scoreboard_bk2_kombi.html.erb` (255 Z →
    new, based on the Karambol scoreboard with BK2 deltas per D-03..D-13)
  - `app/javascript/controllers/bk2_kombi_shot_controller.js` (rewritten
    to target the new bottom-bar DOM)
  - `test/system/bk2_kombi_scoreboard_test.rb` (rewritten to drive the
    new Karambol-parallel layout)
  No `_v1` renaming, no deletion-then-add split — single commit per file
  showing the full redesign diff. Git history is the versioning layer.

### Bug-Fix Scope (UAT-GAP-02..GAP-05)

- **D-18:** **All four UAT-GAP bugs are fixed in Phase 38.2**, because the
  affected files are being rewritten anyway:
  - **UAT-GAP-02** — Alpine `x-data` scope bug in
    `scoreboard_free_game_karambol_new.html.erb`: hidden form inputs at
    lines 179–187 are **outside** the `<div x-data>` scope at line 193,
    so `:value="bk2_selected ? ..."` bindings never evaluate. Fix: move
    the hidden inputs inside the `x-data` scope (or lift the `x-data`
    onto an outer wrapper that covers both inputs and radio selects).
  - **UAT-GAP-03** — i18n phase-key guard in the scoreboard partial:
    `t("table_monitor.bk2_kombi.phase.#{current_phase}", default: ...)`
    returns the parent hash as rendered text when `current_phase` is
    blank. Fix: guard with `if current_phase.present?` or provide an
    explicit blank fallback.
  - **UAT-GAP-04** — missing Home/Cancel nav in BK2 scoreboard:
    automatically resolved by D-01 (1:1 copy of `_show.html.erb`).
  - **UAT-GAP-05** — `bk2_state` initialization fallback: `TableMonitor`
    records with `data["free_game_form"] = "bk2_kombi"` but no
    `data["bk2_state"]` (from pre-Plan-06 test state or manual data
    manipulation) render the scoreboard with all-zero defaults
    (Satz 0, Ziel 0, etc.) and crash the reflex. Fix: on scoreboard
    render, if `free_game_form == "bk2_kombi"` but `bk2_state` is
    empty, either auto-initialize from `bk2_options` OR render a
    clear "Spiel nicht initialisiert — bitte neu starten" banner with
    the Home/Cancel nav active.

### Service-Layer Adaptations (Minimal)

- **D-19:** `Bk2Kombi::AdvanceMatchState` gains **optional** state fields
  to support the new phase-sensitive badge (D-11):
  - `shots_left_in_turn` — already exists; kept as-is for DZ sets.
  - `innings_left_in_set` — **new** field, tracks remaining Aufnahmen
    for Serienspiel sets. Decrements on each Aufnahmewechsel. Default
    configured via `bk2_options.serienspiel_max_innings` (default 5).
  - `first_set_mode` — **new** field in `bk2_state`, derived from the
    start-game payload (D-14). Drives Satz-1/2/3 mode assignment.
  - Scoring math (`Bk2Kombi::ScoreShot`) **unchanged**.
- **D-20:** `bk2_options` schema extension:
  - `set_target_points` — existing.
  - `direkter_zweikampf_max_shots_per_turn` — **new**, default 2, bonus
    rules in service logic.
  - `serienspiel_max_innings_per_set` — **new**, default 5, ausschreibungs-
    abhängig.
  - `first_set_mode` — **new**, `"direkter_zweikampf"` | `"serienspiel"`,
    from start-game payload.

### Claude's Discretion

- Exact Tailwind classes to match Karambol's visual weight on the new
  bottom-bar rows.
- Stimulus controller internals (debounce timing, event dispatch, target
  naming).
- System test fixture strategy (FactoryBot vs. inline TableMonitor setup).
- i18n key additions for new labels (`*.phase_chip.direkter_zweikampf`,
  `*.phase_chip.serienspiel`, `*.shots_left_label`, `*.innings_left_label`,
  `*.first_set_mode.button_direkter_zweikampf`, etc.) — structure follows
  existing `table_monitor.bk2_kombi.*` convention.
- Whether D-01's "1:1 structural copy" is done as a physical `cp` +
  modify, or as a newly-authored file following the Karambol template —
  functional equivalence is the only requirement.
