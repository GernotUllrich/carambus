## Terminal-state guard

Bar wrapper lives behind `show_bottom_bar = !match_finished && !table_monitor.bk2_state_uninitialized? && current_phase.present?`. Render matrix:

| Scenario                                                  | `show_bottom_bar` |
| --------------------------------------------------------- | ----------------- |
| Normal BK2 match in a DZ set with active phase            | true              |
| Normal BK2 match in a SP set with active phase            | true              |
| `match_finished == true`                                  | false             |
| `bk2_state_uninitialized? == true` (pre-Plan-01 fallback) | false             |
| `current_phase` blank (defensive; should never occur after init) | false       |

When `show_bottom_bar` is false, the scoreboard renders the 3-pane row only — which for `match_finished` displays the green banner at the top, and for `bk2_state_uninitialized?` triggers the Plan 03 fallback branch in `_show_bk2_kombi.html.erb`.
