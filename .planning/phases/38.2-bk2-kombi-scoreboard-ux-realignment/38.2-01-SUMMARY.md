### app/services/bk2_kombi/advance_match_state.rb (+41/-6 net)

- Added 3 new constants: `DEFAULT_DZ_MAX_SHOTS_PER_TURN = 2`, `DEFAULT_SP_MAX_INNINGS_PER_SET = 5`, `DEFAULT_FIRST_SET_MODE = "direkter_zweikampf"`.
- Added 4 new private helpers: `derive_dz_max_shots`, `derive_sp_max_innings`, `derive_first_set_mode` (whitelisted), `phase_for_set(set_number, first_set_mode)`.
- Rewrote `init_state_if_missing!`: now seeds `current_phase`, `first_set_mode`, `shots_left_in_turn`, `innings_left_in_set` from `bk2_options` (phase-aware — DZ first seeds shots_left, SP first seeds innings_left).

- **No hard-coded `2` or `5` literals remain in `init_state_if_missing!` or `close_set_if_reached!`.**

## Deviations from Plan

**None — plan executed exactly as written, with two minor planned-ahead accommodations:**

1. **T4b test setup adjustment** — during TDD RED the plan's suggested test-setup pattern (initial `fallen_pins: 0` shot + force `playera` to 47 + 5-pin shot) did not close the set when `first_set_mode == "serienspiel"` because the first shot in SP ends the Aufnahme and swaps players, so the 5-pin shot scores for `playerb` who starts from 0. Adjusted to use a non-turn-ending initial shot (`fallen_pins: 2`) so `playera` remains at table. Assertions unchanged. Documented inline.

2. **Test 3 (pre-existing characterisation test) updated** — the plan's Task 1 <behavior> spec says "Existing DZ bonus-shot behavior unchanged" and "Existing tests must still pass", but Test 3 asserted `current_phase == "direkter_zweikampf"` on set 2 — which was 38.1's behaviour (no phase alternation) and is explicitly replaced by D-14. Updated the assertion to reflect D-14 semantics (set 2 = flipped phase). Rationale documented inline in the test.

Neither is a Rule 1/2/3 deviation — both are directly implied by the plan's D-14 target semantics.
