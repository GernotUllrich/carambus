## Deviations from Plan

**None of the three auto-fix rules (Rules 1-3) triggered — test-side fixes only:**

1. **Test data stubs added** — the plan's action block specified only `bk2_state`, `bk2_options`, `current_kickoff_player`, `free_game_form`. After the first test run showed `_menu.html.erb` (rendered by Plan 03 D-01 Karambol-parallel scoreboard) crashing on `can_undo?` because our data had no `current_inning`/`playera`/`playerb` keys, I added minimal stubs to `initial_bk2_data`. This is pure test-layer scaffolding — does not affect any BK2-specific assertion. Documented in the helper method's header comment. Not a Rule 2 deviation (no production code changed).

2. **assert_text signature adjustment** — Capybara's `assert_text(type, text)` treats the first positional arg as a type selector when a second arg is given. The plan's suggested skeleton passed the banner title string + an explanatory message; that raises `ArgumentError: '...' is not a valid type for a text query`. Fixed by dropping the message arg; explanation moved to a code comment.

3. **assert_selector :message kwarg → has_selector? + assert pattern** — Capybara's `assert_selector` kwargs whitelist excludes `:message`. The plan's suggested skeleton passed `message: "expected Stimulus target..."`. Fixed by using `page.has_selector?` + Minitest `assert(..., msg)` to preserve per-target failure context.

4. **foul_code = :wrong_ball (symbol, not string)** — `Bk2Kombi::ScoreShot` uses `deep_symbolize_keys` on the payload (keys only — not values), then case-dispatches on `foul_code` as a symbol. The plan's suggested skeleton showed `foul_code: "wrong_ball"` (string), which falls through the case to `:other` (returns `-1` not `-6`). Fixed by using `:wrong_ball` — which also mirrors the real `TableMonitorReflex#bk2_kombi_submit_shot` payload shape. This is consistent with existing 38.1 `score_shot_test.rb` which passes symbols throughout.

All four adjustments are plan-skeleton refinements, not production code changes. No Rule 1/2/3 auto-fix applied.
