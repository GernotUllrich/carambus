After this plan, the 5 BK-* Discipline records carry the following `data` JSON:

| Discipline | id | free_game_form | ballziel_choices | nachstoss_allowed | multiset_components | allow_negative_score_input | negative_credits_opponent |
|---|---|---|---|---|---|---|---|
| BK2-Kombi | 107 | bk2_kombi | [50,60,70] | true | ["bk_2plus","bk_2"] | **— not set —** | **— not set —** |

**BK2-Kombi (id 107) explicitly omits both BK-param keys** — per D-08, BkParamResolver looks up the per-set `effective_discipline` (bk_2plus or bk_2 per set index) via `multiset_components[(set_index - 1) % 2]` and reads THAT Discipline's `data["allow_negative_score_input"]` / `data["negative_credits_opponent"]`.

## Decisions Made

- **BK2-Kombi explicitly omits the two BK-param keys** (D-08): seed script's idempotency guard actively `delete`s them if a previous run accidentally wrote them. This makes the contract self-healing: a future operator can re-run the seed any time without worrying about state drift.
- **Test fixture mirrors seed output verbatim**: keeps unit test reads consistent with what local servers receive after Version sync. Avoids decision drift between code-level test data and runtime data.
- **needs_update boolean checks 3 keys** (ballziel_choices, nachstoss_allowed, multiset_components): minimal sufficient set. Adding more keys to the check would catch hypothetical drift but isn't necessary — the data structure on BK2-Kombi only legitimately holds these 3 + free_game_form.
- **Standardrb auto-fix accepted**: 3 layout offenses (1 pre-existing magic-comment, 2 new multiline-indentation) auto-fixed via `standardrb --fix`. All cosmetic, no behavior change.
