**Test method for ISOL-01 + ISOL-04:**

Name: `test "ISOL-01 + ISOL-04: morph path isolation — scoreboard A unchanged when table B state changes, console.warn proves filter ran"`

Flow:
1. `in_session(:scoreboard_a)` — visit TM-A scoreboard, assert `#full_screen_table_monitor_#{@tm_a.id}` present, call `wait_for_actioncable_connection`. Then install a console.warn interceptor via `page.execute_script` to capture the SCOREBOARD MIX-UP PREVENTED message (use the DOM marker fallback from RESEARCH Pitfall 5 in case Selenium logs API is unavailable):
```javascript
window._mixupPreventedCount = 0;
const _origWarn = console.warn;
console.warn = function(...args) {
  if (args[0] && String(args[0]).includes("SCOREBOARD MIX-UP PREVENTED")) {
    window._mixupPreventedCount++;
  }
  _origWarn.apply(console, args);
};
```

2. `in_session(:scoreboard_b)` — visit TM-B scoreboard, assert `#full_screen_table_monitor_#{@tm_b.id}` present, call `wait_for_actioncable_connection`.

3. Trigger broadcast for TM-B only:
```ruby
@tm_b.reload
@tm_b.ready!
TableMonitorJob.perform_now(@tm_b.id)
```

4. POSITIVE assertion (Session B receives update per D-05):
```ruby
in_session(:scoreboard_b) do
  assert_selector "#full_screen_table_monitor_#{@tm_b.id}", text: /Frei/i, wait: 10
end
```

5. NEGATIVE assertion — Session A DOM unchanged + ISOL-04 console.warn proof:
```ruby
in_session(:scoreboard_a) do
  # Give broadcast time to arrive and be processed by JS filter
  sleep 2  # Acceptable: asserting absence after giving enough time
  # ISOL-04: console.warn proves the JS filter rejected the broadcast
  count = page.evaluate_script("window._mixupPreventedCount")
  assert count.to_i > 0,
    "Expected JS filter console.warn('SCOREBOARD MIX-UP PREVENTED') for rejected TM-B broadcast " \
    "on scoreboard A, but got 0 occurrences. Verify: (1) Session A subscribed before broadcast, " \
    "(2) broadcast reached client, (3) getPageContext() identified page as 'scoreboard'"
  # ISOL-01: Session A must NOT show TM-B's scoreboard container
  refute_selector "#full_screen_table_monitor_#{@tm_b.id}"
end
```

Important implementation notes:
- Use `frozen_string_literal: true` at top of file
- Use the console.warn override approach (DOM marker fallback) rather than `page.driver.browser.logs.get(:browser)` — the Selenium logs API may not be available in all ChromeDriver versions (see Research Pitfall 5)
- The `sleep 2` in the negative assertion is acceptable because we are asserting absence — there is no DOM element to poll for. The `_mixupPreventedCount` check provides the positive evidence that the broadcast arrived and was filtered.
- All `page` interactions must be inside their respective `in_session` blocks (Research Pitfall 6)
