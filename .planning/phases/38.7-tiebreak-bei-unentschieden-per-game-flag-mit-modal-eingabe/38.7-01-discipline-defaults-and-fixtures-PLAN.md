<objective>
Seed the central `Discipline` records for BK-2 and BK-2kombi with the
`tiebreak_on_draw: true` discipline-level default, and mirror the same shape into
`test/fixtures/disciplines.yml` so downstream plans (03/04/05) can rely on the
discipline-level fall-through layer in their tests.

Purpose: Establishes the bottom layer of the per-game tiebreak hierarchy (D-04, D-06):
Tournament.data → TournamentPlan.executor_params['gN'] → Discipline.data['tiebreak_on_draw']
→ default `false`. Without this, BK-2 / BK-2kombi training matches would silently
default to `false` and re-introduce the legacy deadlock the phase is supposed to fix.

Output: Updated seed script (carambus_api source of truth) + matching test fixture rows.
</objective>

<problem_framing>
Per CONTEXT.md D-06: BK-2 and BK-2kombi default `tiebreak_on_draw: true` because
the SP-phase deadlock (Nachstoss-Spieler erreicht Ballziel) is rule-bound and recurs
in every BK-2 / BK-2kombi-SP match. Karambol/Snooker/Pool/BK-2plus/BK50/BK100 default
`false` (sparse — key is omitted, not explicitly false, so a higher-level Tournament
override of `true` still takes effect via Plan 03's resolver).

Per CONTEXT.md D-14: discipline.data syncs to local servers via the
`Version#update_from_carambus_api` pipeline (38.4 D-04/D-05 already shipped). No
new sync code; the new key just travels along.

Test fixtures must mirror the post-seed state so unit tests in Plans 03/04/05
can read `Discipline.find_by(name: "BK-2").data["tiebreak_on_draw"]` directly
without staging custom records.
</problem_framing>
