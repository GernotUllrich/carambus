Adds a checkbox "Stechen bei Unentschieden" / "Tiebreak on draw" to the
TournamentMonitor `_form.html.erb`. Default value derives from the bound
TournamentPlan's executor_params['g1']['tiebreak_on_draw'] (resolver Level 2 — so
the operator sees the plan default pre-checked). On form **UPDATE** (not create),
the controller persists the operator's pick onto `Tournament.data['tiebreak_on_draw']`
(resolver Level 1) so Plan 04's existing resolver picks it up at every game's
start_game.

**Auth surface — important:** The existing controller authorization
(`ensure_tournament_director` + `ensure_local_server`) is registered with
`only: %i[show edit update destroy update_games switch_players start_round_games]`.
**`:create` and `:new` are NOT in that list.** Therefore this plan deliberately
gates the persistence of `Tournament.data['tiebreak_on_draw']` to the `:update`
action ONLY. The checkbox renders in `new` for UX continuity, but its value is
ignored on `:create`. Operator workflow: create the TournamentMonitor first
(unauthenticated-friendly path stays unchanged), then immediately edit it once to
tick the checkbox. This avoids widening the existing authorization surface.

Purpose: Operators no longer need Administrate or the Rails console to enable
tiebreak per-tournament. The form is the canonical surface; the operator's pick
wins over plan default.

```ruby
def self.derive_tiebreak_required(tournament: nil, tournament_plan: nil, group_no: nil)
  # Level 1: Tournament.data
  t_data = parse_data_hash(tournament&.data)
  return !!t_data["tiebreak_on_draw"] if t_data.key?("tiebreak_on_draw")

  # Level 2: TournamentPlan.executor_params['g{group_no}']
  if tournament_plan && group_no.present?
    plan_params = parse_data_hash(tournament_plan.executor_params)
    group_key = "g#{group_no}"
    if plan_params.is_a?(Hash) && plan_params[group_key].is_a?(Hash) &&
        plan_params[group_key].key?("tiebreak_on_draw")
      return !!plan_params[group_key]["tiebreak_on_draw"]
    end
  end

  # Level 3: default
  false
end
```

Tournament.data is what we write. The resolver uses `data.key?("tiebreak_on_draw")`

        ```ruby
            # Phase 38.5 Plan 02 D-04 sparse-override semantics). Full reassignment

          end
        ```
