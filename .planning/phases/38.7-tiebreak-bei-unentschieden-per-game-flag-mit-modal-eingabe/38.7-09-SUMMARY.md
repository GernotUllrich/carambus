## Decisions Made

- **Override placement:** preset override sits AFTER the resolver call (not before). Rationale: in training mode the resolver returns false; a truthy preset must flip it. In tournament mode the resolver returns the Tournament/Plan-level value, which is authoritative — but the preset still wins if explicitly supplied (per the sparse-override convention). The plan-prescribed code already had this structure; documented here for clarity.
- **Hidden field as '1'/'0' string** (not true/false): matches the codebase's existing convention for `allow_follow_up` (line 162 of the partial). Controller normalization handles the string→Bool conversion.
- **`.erb` template edited alongside compiled `.yml`**: the .yml is gitignored. The .erb is the authoritative template; both must be kept in sync per the STATE.md memory entry for Phase 38.4.
