## Decisions Made

- **Explicit-false semantics via `hidden_field_tag` (option B over option A)**: chose to emit `tiebreak_on_draw="0"` when unchecked rather than rely on absent-key sparse semantics. Rationale: sparse-key semantics work today (resolver default-false in training mode = same outcome), but the explicit-false form locks the contract in advance of any future tournament-mode interaction where an explicit operator-pick of "off" must override an upstream truthy resolver. Matches the plan's prescribed code (the second variant in `<action>` Step 3a).
- **Tests live as regression characterization, not RED-then-GREEN**: Plan 09 already shipped the controller + service wiring; the Plan 10 view edit doesn't change test outcomes (tests POST params directly, not via the form). The plan author called this out (`<behavior>` block) — tests are explicit Gap-02 contract guards rather than load-bearing RED gates.
- **Plan 09 wiring untouched**: `app/controllers/table_monitors_controller.rb` and `app/services/table_monitor/game_setup.rb` are byte-identical to their state after `15838186` (Plan 09 GREEN). Verified via `git diff HEAD~2 HEAD app/controllers/table_monitors_controller.rb` (zero diff) — exactly as the plan's AC12 requires.
- **Karambol form chosen for tests over BK-* form**: existing controller tests for the BK-* path require a Discipline.find_by chain that's entangled with the BK-2kombi CLAMP. Karambol path is cleaner for the regression contract and the bake logic is form-agnostic (preset-override block runs regardless of free_game_form). Both BK-* and karambol detail-forms use the same view template (`scoreboard_free_game_karambol_new.html.erb`) so the toggle is visible on both.
