## Decisions Made

- **Inject inside `tiebreak_pick_pending?`, not at 4 call sites**: extend-before-build SKILL — one guard at the natural injection point. Test runs all 4 paths via the existing T1-T9 suite + new G1-G3 suite; no consumer-side changes needed.
- **Persist via `Game#deep_merge_data!`**: Plan 04 + Plan 06 already documented the dirty-tracking footgun — Game's custom `def data` getter returns a freshly-decoded Hash, so direct mutation doesn't trigger `data_will_change!`. `deep_merge_data!` is the canonical write path; reusing it keeps the auto-detect aligned with Plan 04's resolver bake mechanism.
- **`balls_goal.positive?` defensive guard**: prevents zero-zero false positives during early game state when result/balls_goal might both be zero. Plan didn't explicitly call this out, but the 5-condition gate spec in Gap-03 implies "at goal" means a meaningful (positive) goal value.
- **Audit log line BEFORE mutation**: STRIDE T-38.7-11-02 (Repudiation) mitigation — operators cannot deny modal was triggered if the log records the 5-condition snapshot at the moment of detection.
