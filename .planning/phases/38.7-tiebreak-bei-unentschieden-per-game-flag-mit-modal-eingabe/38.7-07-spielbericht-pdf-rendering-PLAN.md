<objective>
Render the tiebreak indicator in the Spielbericht-PDF (`parties/_spielbericht.html.erb`)
when `game.data['ba_results']['TiebreakWinner']` is present. Per CONTEXT.md D-12,
the format is `<score> (Stechen <Player-shortname>)`, e.g. `15:15 (Stechen A)`.

Per CONTEXT.md specifics §"i18n-Notation Spielbericht": the user prefers either
`15:15 (Stechen A)` or `15:15 → A`. We pick the parenthesized "Stechen A" form
because it's clearer to volunteers who may not be familiar with the arrow notation.
The exact label is i18n-driven: `parties.spielbericht.tiebreak_won_by` with a
`%{role}` interpolation.

Output: ~5-line view edit + ~15-line helper method + 1 i18n key × 2 languages + 4
unit tests on the helper.
</objective>

    **Step 1 — Add the helper to `app/helpers/parties_helper.rb`**.

    Append (or insert next to existing methods) this method:

    ```ruby
    # Phase 38.7 Plan 07 D-12 — Tiebreak indicator for Spielbericht-PDF.
    # Returns a localized suffix " (Stechen <PlayerName>)" when the game has a
    # tiebreak winner recorded; empty string otherwise. Defensive against nil
    # game and invalid TiebreakWinner values.
    #
    # Reads game.data['ba_results']['TiebreakWinner']  (1 = playera, 2 = playerb).
    # The integer is written by TableMonitor::ResultRecorder#update_ba_results_with_set_result!
    # (Plan 05) when the operator picks a winner via the protocol_final modal (Plan 06).
    def tiebreak_indicator(game)
      return "" if game.nil?
      tw = game.data&.dig("ba_results", "TiebreakWinner")
      role = case tw.to_i
             when 1 then game.player_a&.shortname.presence || game.player_a&.fullname
             when 2 then game.player_b&.shortname.presence || game.player_b&.fullname
             else nil
             end
      return "" if role.blank?
      " #{t("parties.spielbericht.tiebreak_won_by", role: role)}"
    end
    ```
