### Gap-03 — BK-2kombi BK-2-phase auto-detect

**Symptom:** The BK-2kombi multiset discipline runs a BK-2 component as one of its phases. In that BK-2 phase, when the Anstoss-Spieler reaches `balls_goal` in inning 1 (single-inning win) AND the Nachstoss-Spieler matches `balls_goal` in his Nachstoss-Aufnahme, the result is a tied set — and a tiebreak SHOULD be required by the rules of the discipline regardless of any per-game/preset/tournament flag. Today this case has no automatic detection: an operator who didn't tick the toggle gets no modal.

**Decision (locked, user 2026-04-30):**
> "special case BK-2kombi in BK-2 phase with the special condition on follow up only when player hits balls_goal in first inning and opponent reaches balls_goal as well"

**Required behavior:**
- In BK-2kombi when the active multiset phase is BK-2 (`bk2_state` indicates BK-2 component), and the set ends with both players at balls_goal in single-inning + Nachstoss-Aufnahme parity, set `Game.data['tiebreak_required'] = true` at the moment of detection (NOT at start_game — this is a runtime override based on actual play state)
- Detection runs in or near `TableMonitor::ResultRecorder` (current location of tiebreak detection wired in plan 05) — it must trigger BEFORE the existing `tiebreak_pick_pending?` check so the marker switch fires
- The auto-detect overrides any pre-baked `tiebreak_required=false`; explicit operator/preset/tournament `false` does NOT defeat it (rule of the discipline, not configurable)
- Trigger conditions:
  - `bk2_state` indicates BK-2 phase active
  - Anstoss reached balls_goal in 1 inning (first inning)
  - Nachstoss matched balls_goal in his Nachstoss-Aufnahme (inning 2 in his frame)
  - Both scores equal at end of set

**Files to touch:**
- `app/services/table_monitor/result_recorder.rb` — BK-2kombi-aware tiebreak detection guard injected before / inside `tiebreak_pick_pending?`
- `test/services/table_monitor/result_recorder_test.rb` — characterization tests for the auto-detect path
- Possibly `app/services/bk2/advance_match_state.rb` if state inspection helpers live there

status: resolved
fix_plan: 38.7-11 (gap_closure: true) — landed 2026-04-30 (commits dd97be5c / f3915960 / 8773f815); auto-detect helper runs as first line of tiebreak_pick_pending?
