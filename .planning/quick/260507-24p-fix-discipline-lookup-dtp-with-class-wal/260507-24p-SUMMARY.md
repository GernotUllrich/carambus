# Quick 260507-24p: Fix Discipline#lookup_dtp_with_class_walk seedings filter Summary

Smart-fallback seedings filter in `Discipline#lookup_dtp_with_class_walk` closes Phase 39 UAT Gap-01 and unblocks the verification modal for production tournaments with mixed local + global synced seedings.

## What Changed

### Production code (`app/models/discipline.rb`)

**1-line filter patch** at `lookup_dtp_with_class_walk` line 93:
```ruby
# Before:
.where(players: tournament.seedings.count)
# After:
.where(players: effective_player_count(tournament))
```

**New private helper** appended directly below `lookup_dtp_with_class_walk` (still inside the existing `private` section, before `range_from_canonical`):
```ruby
def effective_player_count(tournament)
  local_count = tournament.seedings.where("seedings.id >= ?", Seeding::MIN_ID).count
  return local_count if local_count > 0

  tournament.seedings.where("seedings.id < ?", Seeding::MIN_ID).count
end
```

The helper mirrors the canonical smart-fallback pattern from `tournament_cc.rb:286`:
- **Local server case** (operator-created tournament with id ≥ MIN_ID seedings + synced global aliases): use the local-only count → DTP lookup matches the correct row.
- **Central API server case** (the API IS the source — all seedings are id < MIN_ID by construction): fall back to the global-only count → DTP lookup still works.

### Test fixture (`test/fixtures/seedings.yml`)

**3 globally-id'd seedings** appended (IDs 9_000_010..9_000_012, all `< Seeding::MIN_ID`, all attached to `tournament_id: 50_000_200` = `local_fpk_class1`). They represent central-API-synced "global" seedings that arrive on local servers via the Version sync mechanism alongside operator-created local seedings.

After the fixture extension: `tournaments(:local_fpk_class1).seedings.count == 8` (5 local + 3 global). Without the fix, `parameter_ranges` would return `{}` because no DTP row matches `players=8`. With the fix, the smart fallback uses local-only count of 5 → DTP row `fpk_t04_5_class1` matches → returns `{balls_goal: 187..250, innings_goal: 11..15}`.

### Test coverage (`test/models/discipline_test.rb`)

**Test 1 (Gap-01 mixed-seedings local-path)** — RED until the model fix landed; pins the fixture-shape precondition (5 local + 3 global = 8 total) and asserts the resolver returns `players=5` Range, not `players=8`.

**Test 2 (Gap-01 fallback central-API-path)** — Builds a transient tournament + 5 global seedings (no fixture changes); pins the else-branch of the smart fallback. Defense-in-depth guard against future refactors that hardcode the local-only filter.
