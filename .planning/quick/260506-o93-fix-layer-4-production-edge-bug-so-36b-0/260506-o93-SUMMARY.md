## Why the Form Keeps `0` and `999`

The `tournament_monitor` form's `sets_to_play` and `sets_to_win` selects intentionally include sentinel values that fall OUTSIDE `UI_07_SHARED_RANGES`:

- **`0` = "-"** (not yet set / single-set mode). Default state before the operator picks a sets-config. Used for single-set tournaments (Test 4's exact scenario) and as the initial state for any new tournament.
- **`999` = "no limit"** (only for `sets_to_play`). User-meaningful "play until first to N sets wins" without a hard cap.

The fix correctly distinguishes "user picked a deliberate sentinel" (skip range check) from "user typed a typo" (still flag). Defense-in-depth verified by Test 6 of the new regression test: `sets_to_win=999` is **not** a form option (only `0`, `2`, `3`, `4`), so submitting `999` for `sets_to_win` IS a real out-of-range value and remains flagged. The exemption is field-scoped, not blanket.
