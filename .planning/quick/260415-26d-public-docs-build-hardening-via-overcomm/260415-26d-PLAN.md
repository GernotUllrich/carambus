<objective>
Install a local overcommit pre-commit hook that prevents developers from ever committing `docs/**/*.md` source edits without regenerating `public/docs/**/*` in the same commit. This structurally fixes the stale-docs class of bug that caused G-02 in v7.0 UAT (commit `7cf16114` rebuilt 257 files after 4 weeks of drift).

Purpose: Make "I forgot to run mkdocs build" impossible on a workstation with the hook active. The Rails server at `/docs/` serves the tracked `public/docs/` tree, so any divergence between `docs/` (source) and `public/docs/` (generated) silently ships stale content.

Output: `.overcommit.yml` + dedicated hook script + bilingual developer docs + working hook on the developer's machine after `overcommit --install && overcommit --sign`.
</objective>

overcommit CustomHook shape (minimal relevant subset):
```yaml
# .overcommit.yml
PreCommit:
  MkDocsBuild:
    enabled: true
    required: true
    description: 'Regenerate public/docs/ when docs/**/*.md changes'
    include:
      - 'docs/**/*.md'
    command: ['bin/overcommit/mkdocs-build-on-docs-change']
```

Key overcommit facts the executor must know:
- `include:` is a list of file glob patterns. The hook runs ONLY when the staged set intersects the include globs. No files matching → hook is skipped entirely.
- Files `git add`ed inside a hook script ARE folded into the in-progress commit.
- First-time install on a fresh clone requires: `bundle exec overcommit --install` (wires `.git/hooks/*`) then `bundle exec overcommit --sign` (signs the config — overcommit refuses to run an unsigned `.overcommit.yml` as a safety measure).
- `SKIP=MkDocsBuild git commit -m ...` bypasses the hook for a single commit (documented escape hatch).

  <action>
Create `.overcommit.yml` in repo root with a single PreCommit CustomHook named `MkDocsBuild`:

```yaml
# .overcommit.yml — Managed by bundle exec overcommit --install
# Activation: bundle exec overcommit --install && bundle exec overcommit --sign
# Bypass a single commit: SKIP=MkDocsBuild git commit ...
PreCommit:
  ALL:
    quiet: false

  MkDocsBuild:
    enabled: true
    required: true
    description: 'Rebuild public/docs/ when docs/**/*.md changes (G-02 hardening)'
    include:
      - 'docs/**/*.md'
    command: ['bin/overcommit/mkdocs-build-on-docs-change']
```

Leave all other overcommit default hooks DISABLED in this config (do not enable RuboCop, Brakeman, etc. — the project already runs those through other paths and enabling them here would change the scope of this task and could block unrelated commits).

Create `bin/overcommit/mkdocs-build-on-docs-change` (executable bash script). Requirements:

1. Shebang `#!/usr/bin/env bash` + `set -euo pipefail`.
2. Header comment: what it does, when it fires, how to bypass.
3. Belt + suspenders check: re-detect whether any staged file matches `docs/.*\.md$` via `git diff --cached --name-only --diff-filter=ACMR`. If zero matches → echo "[MkDocsBuild] no docs/**/*.md in staged set, skipping" and `exit 0`. (The include filter should already handle this, but defending against an empty filter is cheap and explicit.)
4. Pre-flight: check `command -v mkdocs >/dev/null 2>&1`. On miss, print:
   ```
   [MkDocsBuild] ERROR: mkdocs CLI not found.
   [MkDocsBuild] Install with:
   [MkDocsBuild]   pip install mkdocs-material mkdocs-static-i18n pymdown-extensions
   [MkDocsBuild] Or bypass this hook for one commit: SKIP=MkDocsBuild git commit ...
   ```
   and `exit 1`.
5. Run `bin/rails mkdocs:build`. If it exits non-zero, print `[MkDocsBuild] ERROR: bin/rails mkdocs:build failed — see output above. Aborting commit.` and `exit 1`. The rake task already prints its own error detail, so do NOT swallow its stdout/stderr.
6. On success, run `git add public/docs/` to fold the regenerated files into the in-progress commit. Print `[MkDocsBuild] Regenerated public/docs/ and staged changes into current commit.`
7. `exit 0`.

After writing the script, `chmod +x bin/overcommit/mkdocs-build-on-docs-change`.

DO NOT modify `lib/tasks/mkdocs.rake`. DO NOT touch any file under `.git/hooks/`. DO NOT enable any overcommit default hook beyond MkDocsBuild.
