# Quick Task 260415-26d: public/docs build hardening via overcommit — Summary

Installed an overcommit pre-commit hook (`MkDocsBuild`) that structurally prevents the drift class of bug that caused v7.0 UAT gap G-02. When any `docs/**/*.md` file is in the staged set, the hook runs `bin/rails mkdocs:build` and folds the regenerated `public/docs/` tree into the in-progress commit atomically. Commits that don't touch docs see zero overhead.

## Deviations from Plan

### Rule 3 — Blocking issue: overcommit default hooks

**Found during:** Task 2 Smoke Test A.

**Issue:** Overcommit ships with several built-in hooks auto-enabled (`AuthorEmail`, `AuthorName`, `BrokenSymlinks`, `CaseConflicts`, `MergeConflicts`). The plan said "Do NOT enable any overcommit default hook beyond MkDocsBuild", but overcommit auto-enables them unless you explicitly disable. `BrokenSymlinks` immediately failed on the pre-existing intentional external symlinks `.env → docker/development/api-server/env.development.api-server` and `ansible → /Users/gullrich/DEV/ansible`, which would have blocked every future commit on the repo.

**Fix:** Added explicit `enabled: false` entries for all five default pre-commit hooks in `.overcommit.yml` with a block comment explaining the rationale. Smoke Test A then passed with only `MkDocsBuild` reporting.

**Files modified:** `.overcommit.yml` (additional 15 lines disabling defaults)

**Commit:** `912bf72a` (folded into the final atomic commit)

### No other deviations

The plan's mkdocs CLI absence fallback path was never needed — mkdocs is installed on this workstation.
