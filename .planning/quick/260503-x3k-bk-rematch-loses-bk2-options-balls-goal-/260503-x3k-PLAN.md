<objective>
Fix BK-rematch losing `bk2_options.balls_goal` so BK-2 / BK-2plus rematches keep showing the configured Ballziel (e.g. 70) instead of "?" or 50.

Purpose: User report 2026-05-03 — after a BK-2/BK-2plus single-set match ends, pressing "Nächstes Spiel" starts a new game with swapped Anstoß (correct) but the score panel renders "?" or 50 as Ballziel. Root cause: `TableMonitor#revert_players` builds an options hash with 17 keys for `start_game` but omits `bk2_options`. Downstream, `GameSetup#build_result_hash` writes `"bk2_options" => @options["bk2_options"]` into the new game's data; with nil input, the new game has no `bk2_options`, and `_player_score_panel.html.erb` falls back to `data.dig("bk2_options", "balls_goal")` which is now nil → renders "?".

Output: One-line fix in `revert_players` plus a RED-then-GREEN regression test. The underlying game logic (`data["playera"]["balls_goal"]`) is already preserved correctly — only the BK-family scoreboard's `bk2_set_target` path needs `bk2_options` to round-trip.
</objective>
