# Phase quick-260505-0b5: CR-02 Sentinel Restore — Narrow-Scoped Per-TM Summary

Per-TM thread-local sentinel restored to `TableMonitor#advance_tournament_round_if_present`, narrow-scoped to `self.id` so legitimate cross-TM cascades during `TournamentMonitorState#finalize_round` are preserved while same-TM re-entry short-circuits — closes the live 2026-05-05 SystemStackError on tournament-finals "Nächstes Spiel".

## What Was Done

### Task 1 — RED tests (commit `ff711fee`)

Added 4 tests to `test/services/tournament_monitor/result_processor_test.rb` (+161 LOC):

1. **`CR-02 tournament-finals close_match cascade does NOT recurse infinitely (regression)`** — Service-level dispatch through the AASM after-callback chain. Builds 2 TableMonitors (state `:final_match_score`, non-blank game.data) attached to a real `@tournament` from `KoTournamentTestHelper`; stubs `populate_tables / incr_current_round! / start_playing_groups! / group_phase_finished? / all_table_monitors_finished?` on the TournamentMonitor and `TournamentMonitorUpdateResultsJob.perform_later / TournamentStatusUpdateJob.perform_later` on the job classes; calls `outer_tm.close_match!`; asserts `assert_nothing_raised` under `Timeout.timeout(10)`. Without the sentinel, this raised SystemStackError after recursive savepoint allocation (live incident reproduction).

2. **`CR-02 advance_tournament_round_if_present short-circuits when sentinel matches self.id`** — Pure unit test. In-memory TableMonitor with `id` stubbed to 99; `Thread.current[:_advancing_round_for_tm] = 99`; `TournamentMonitor::ResultProcessor.stub :new, ->(_) { raise }` proves the delegate is NOT invoked.

3. **`CR-02 advance_tournament_round_if_present DOES run when sentinel is set for a DIFFERENT TM`** — Pure unit test. TM `id` 99; sentinel primed to 42 (different); spy on `advance_round_after_match_close` proves the delegate IS invoked. Locks the per-TM scoping contract.

4. **`CR-02 advance_tournament_round_if_present clears sentinel via ensure even on exception`** — Mirror of original commit 37796f7d test. Sentinel must be `nil` after the delegate raises — verified via `assert_raises(StandardError) { ... }` followed by `assert_nil Thread.current[:_advancing_round_for_tm]`.

RED state confirmed at this commit:
- Test 1: `SystemStackError: 9318 -> 176` (real recursion in `log_state_transition`)
- Test 2: `RuntimeError: ResultProcessor.new must NOT be invoked when sentinel matches self.id` (proves no guard exists)
- Tests 3 + 4: pass vacuously (no guard yet); contract-locks for the GREEN commit.

### Task 2 — GREEN sentinel guard (commit `1709938e`)

Edited `app/models/table_monitor.rb#advance_tournament_round_if_present` (+14 LOC including doc-block):

```ruby
def advance_tournament_round_if_present
  return if tournament_monitor.blank?
  return unless tournament_monitor.is_a?(TournamentMonitor)
  if Thread.current[:_advancing_round_for_tm] == self.id
    Rails.logger.info "[advance_tournament_round_if_present] m6[#{id}] sentinel SHORT-CIRCUIT (CR-02 per-TM re-entry guard)"
    return
  end
  Thread.current[:_advancing_round_for_tm] = self.id
  Rails.logger.info "[advance_tournament_round_if_present] m6[#{id}] delegating to ResultProcessor#advance_round_after_match_close"
  TournamentMonitor::ResultProcessor.new(tournament_monitor).advance_round_after_match_close(self)
rescue StandardError => e
  Rails.logger.error "[advance_tournament_round_if_present] m6[#{id}] ERROR: #{e}, #{e.backtrace&.first(3)&.join(" <- ")}"
  raise
ensure
  Thread.current[:_advancing_round_for_tm] = nil if Thread.current[:_advancing_round_for_tm] == self.id
end
```

Doc-block above the method names:
- commit 37796f7d (original CR-02)
- commit 0dbe45c4 (wrong revert 2026-05-01)
- quick task 260505-0b5 (this restore — narrow-scoped per-TM)
- the recursion path, end-to-end
- the per-TM scoping rationale (cross-TM cascade preservation)
- SKILL extend-before-build attestation

**Key design point — per-TM vs boolean:** Sentinel value is `self.id` (Integer or `nil` for unsaved TMs), NOT `true`. The original 37796f7d implementation used a boolean which would have over-blocked legitimate cross-TM cascades. With `== self.id`, finalize_round's loop calling `tabmon.close_match!` on a DIFFERENT TM still runs that TM's after-callback normally; only re-entry for the SAME outer TM short-circuits.

**Root cause:** Recursion path
```
TableMonitor#close_match! (AASM)
  → after-callback advance_tournament_round_if_present
  → ResultProcessor#advance_round_after_match_close
  → @tournament_monitor.finalize_round (lib/tournament_monitor_state.rb:54)
  → loops table_monitors, calls tabmon.close_match! per TM
  → re-enters advance_tournament_round_if_present for SAME outer TM → ⤺
```

The original sentinel from commit `37796f7d` (Phase 38.8 CR-02 fix) prevented this. It was reverted by `0dbe45c4` on 2026-05-01 on the basis of BK-2 single-set training-mode evidence that did not exercise the recursive path. The sentinel is now restored, narrow-scoped per-TM.

## Deviations from Plan

**1. [Rule 1 — Bug] Plan-prescribed `location_id` attribute on TableMonitor does not exist**
- **Found during:** Task 1 RED run
- **Issue:** Plan-prescribed `TableMonitor.create!(..., location_id: ...)` raised `ActiveModel::UnknownAttributeError` — the `table_monitors` schema has no `location_id` column (organizer/Location coupling is via Tournament + cattr_accessor :location, not a column on TM).
- **Fix:** Removed `location_id:` from the `create!` call. The test does not need the location FK to exercise the recursion path — only `tournament_monitor`, `game`, `state`, `data` are load-bearing.
- **Files modified:** test/services/tournament_monitor/result_processor_test.rb
- **Commit:** ff711fee (rolled into Task 1 RED commit)

**2. [Rule 1 — Bug] Plan-prescribed `update_columns(data: ...to_json)` clashes with `serialize :data, coder: JSON`**
- **Found during:** Task 1 RED run
- **Issue:** Game has `serialize :data, coder: JSON, type: Hash`. `update_columns` bypasses ActiveRecord callbacks but NOT serializers; passing a String raised `ActiveRecord::SerializationTypeMismatch: was supposed to be a Hash, but was a String`.
- **Fix:** Pass a Hash literal directly: `game.update_columns(data: { "playera" => { "result" => 10 }, ... })` — the JSON serializer round-trips it into TEXT storage automatically.
- **Files modified:** test/services/tournament_monitor/result_processor_test.rb
- **Commit:** ff711fee (rolled into Task 1 RED commit)

**3. [Plan footnote] Tests 3 and 4 RED state is "vacuous-pass", not assertion-fail**

Plan §Task 1 step 4 actually anticipates this: "Test 1 should fail with SystemStackError; Tests 2-4 should fail with the spy assertion" was a worst-case prediction but the tests were also designed as contract locks. Tests 3 and 4 staying GREEN through Task 2 is exactly the desired contract behavior.
