Create test/controllers/tournaments_controller_test.rb using ActionDispatch::IntegrationTest (matching the table_monitors_controller_test.rb pattern).

Structure the test class with:

1. **Setup/teardown:**
   - `@tournament = tournaments(:local)` (id >= 50M, writeable)
   - `@user = users(:one)` for basic user
   - `@club_admin = users(:club_admin)` for admin-gated actions
   - `@original_api_url = Carambus.config.carambus_api_url` saved in setup
   - Restore `Carambus.config.carambus_api_url = @original_api_url` in teardown
   - `sign_in @user` as default

2. **Unauthenticated access test:**
   - Sign out, GET tournaments_url, assert redirect to sign-in page (Devise default)

3. **Read-only GET actions (no local server required):**
   - `index`: GET tournaments_url — assert_response :success
   - `show`: GET tournament_url(@tournament) — assert_response :success (may succeed or redirect depending on tournament state; use `assert_includes [200, 302], response.status`)
   - `tournament_monitor`: GET tournament_monitor_tournament_url(@tournament) — assert_response :redirect (redirects to tournament_monitor_path)
   - `test_tournament_status_update`: POST test_tournament_status_update_tournament_url(@tournament) — assert_response :redirect (enqueues job, redirects)

4. **Local-server-required GET actions:**
   For each: test BOTH paths:
   a. Without local server (default API context): assert_redirected_to tournaments_path
   b. With local server (`Carambus.config.carambus_api_url = "http://local.test"`): assert_response :success or :redirect (action-specific)

   Actions to test:
   - `edit_games`: GET — needs route check; if no custom route, skip
   - `new`: GET new_tournament_url
   - `edit`: GET edit_tournament_url(@tournament)
   - `finalize_modus`: GET finalize_modus_tournament_url(@tournament) — may redirect if tournament not in correct state
   - `define_participants`: GET define_participants_tournament_url(@tournament)
   - `new_team`: GET new_team_tournament_url(@tournament)
   - `compare_seedings`: GET compare_seedings_tournament_url(@tournament)
   - `parse_invitation`: GET parse_invitation_tournament_url(@tournament) — may redirect if no invitation

Note: Some GET actions may fail with 500 if fixtures don't have required associations. In those cases, use `assert_includes [200, 302, 500], response.status` and add a comment explaining the fixture dependency. The goal is to verify the guard behavior (API server redirects, local server passes through), not the full action logic.

For actions that consistently error due to missing fixture data (e.g., finalize_modus needs discipline_tournament_plan), test only the API-server-redirect path and add a comment documenting why the local-server path is not tested.
