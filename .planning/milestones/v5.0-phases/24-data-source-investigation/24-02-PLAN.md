**Cuesco Inspection (INVEST-02 — highest priority per D-02):**

1. Open Chrome/Firefox DevTools (F12) -> Network tab -> Filter by "XHR" or "Fetch"
2. Navigate to `http://umb.cuesco.net/` (note: HTTP, not HTTPS)
   - If ECONNREFUSED: try `https://cuesco.eu` instead, find a live or recent tournament
3. Click on a live or recent tournament link
4. Wait for match data to appear (player names, scores)
5. In the Network tab, look for any XHR/Fetch requests that return JSON
6. For each JSON-returning request, copy:
   - The full URL
   - The HTTP method (GET/POST)
   - The request headers
   - The first 50 lines of the response body
7. Look specifically for endpoints returning match arrays with fields like: player names, scores, round/group data, match numbers

**SoopLive Inspection (INVEST-03 — only if Plan 01 did NOT find a JSON API):**

If Plan 01 SoopLive probes all returned HTML or 404, perform browser inspection:

1. Open Chrome DevTools -> Network tab -> Filter "Fetch/XHR"
2. Navigate to `https://billiards.sooplive.com/schedule/127?sub1=result`
3. Wait for match results to load (player names and scores should appear)
4. Find the XHR/Fetch request that populated the match list
5. Copy the exact URL and first 20 lines of the JSON response
6. Check: do `data-seq` values in the rendered page correspond to VOD IDs? Try opening `https://vod.sooplive.com/player/[data-seq-value]` to verify.

If Plan 01 already found a JSON API for SoopLive, skip this inspection.

**Report your findings here.** Include:
- For Cuesco: Was the site accessible? Were JSON endpoints found? What URL pattern? Sample data fields?
- For SoopLive (if inspected): What was the API URL? What fields were in the response? Did data-seq map to VOD IDs?
- If either site was inaccessible, note that.
