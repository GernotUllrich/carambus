# Requirements: Carambus API v7.1 — ClubCloud Integration Redesign

**Status:** SKELETON — produced during Phase 36c (2026-04-14) as v7.1 groundwork
**Core Value:** Carambus becomes the authoritative tournament workflow UI for events backed by ClubCloud, with automatic finalization and ranking calculation that today is handled manually in CC.

> This document is a **skeleton** written ahead of v7.1. It captures the
> scope, rationale, and known requirements so that v7.1 discuss-phase /
> plan-phase can start from an informed baseline rather than from a blank
> page. Open questions are marked **[TBD: v7.1]** and should be resolved
> during the v7.1 discuss-phase.

## Context

v7.0 Phase 36 (Turnierverwaltung doc review + UI cleanup) exposed three
large gaps in the Carambus ↔ ClubCloud integration that the v7.0 scope
could not absorb:

1. **Endrangliste (final ranking) is computed manually in ClubCloud today.**
   Carambus knows the full game tree but does not produce the final
   ranking in a form that CC accepts, so a human re-enters it.
   (v7.0 finding F-36-34.)
2. **Teilnehmerliste (participant list) finalization via the CC API is
   not implemented.** Both upload paths (per-game and end-of-tournament
   CSV) require the participant list to be "finalized" in CC first —
   Carambus can read the list but cannot flip the finalization bit.
   (v7.0 finding F-36-23.)
3. **Credential delegation for Club-Sportwart rights is organizational,
   not implemented.** Only certain CC roles can add missing players
   mid-tournament; those people aren't always present at the venue.
   (v7.0 finding F-36-23.)

Source: `.planning/v7.0-scope-evolution.md` §"New code feature gaps".

## v7.1 Requirements

Each requirement maps to one or more phases. IDs use the `CCI-` prefix
(ClubCloud Integration) to distinguish from v7.0 requirements.

### Endrangliste (final ranking) automatic calculation

- [ ] **CCI-01**: Carambus computes the final-ranking (Endrangliste) for
  every supported tournament mode (KO, round-robin, group+finals, etc.)
  in a data structure that matches the CC Endrangliste schema — source
  of truth lives in Carambus, CC is the consumer.
- [ ] **CCI-02**: A preview view renders the computed Endrangliste before
  upload so the Turnierleiter can verify and optionally correct it
  (tiebreaker manual override, Shootout resolution, etc.).
- [ ] **CCI-03**: Endrangliste upload uses the existing CC API surface
  (same auth as per-game upload) — no new credentials or roles are
  required for the upload step itself.

### Teilnehmerliste finalization via CC API

- [ ] **CCI-04**: Carambus can mark a Teilnehmerliste as "finalized" in
  CC via API, so per-game upload and CSV upload no longer require the
  Turnierleiter to log into CC in a second browser tab.
- [ ] **CCI-05**: The finalization action is guarded by a preflight check
  that all Carambus-known participants match a CC player record (or are
  explicitly flagged as "missing in CC"). No silent finalization.
- [ ] **CCI-06**: Missing-player flow — when Carambus detects a player
  without a CC record, it either prompts for on-the-fly CC player
  creation (if credentials allow) or marks the player as blocking and
  produces a human-readable report of what needs manual CC work.

### Credentials delegation for Club-Sportwart rights

- [ ] **CCI-07**: A Carambus-side credential profile captures which CC
  role (Club-Sportwart, Region-Admin, etc.) the currently-logged-in user
  has, so the UI can tell the user up-front what they can and cannot do
  in the CC integration.
- [ ] **CCI-08**: Missing-permission flow — when the active user lacks
  the CC role required for a given operation (typically add-missing-
  player), Carambus produces a concrete handoff: which human to call,
  which data to give them, and which resume-point to return to.
- [ ] **CCI-09**: **[TBD: v7.1]** Organizational doc: procedure for Club-
  Sportwart credential delegation (shared account vs. per-tournament
  token vs. impersonation) — decide and document during v7.1 discuss-
  phase with SME input.

## Dependencies

- **v7.0 must be shipped** — specifically Phase 36a (Turnierverwaltung
  Doc Accuracy) must be complete because the v7.1 work touches the
  same tournament-management walkthrough document.
- **Phase 36b reset/parameter modals** must be stable (they ship with
  v7.0) — v7.1 will build on the same confirmation modal infrastructure.
- **Phase 36c appendix draft** (`.planning/clubcloud-admin-appendix-DRAFT.md`)
  must be reviewed and promoted to `docs/managers/` before v7.1 code
  work begins — or in parallel as a v7.1 Phase 1 task.

## Out of Scope

| Feature | Reason |
|---------|--------|
| Shootout / Stechen support | Separate milestone (see v7.2-REQUIREMENTS.md). |
| Match-Abbruch / Freilos handling | Backlog seed; revisit after v7.1 ships. See `.planning/seeds/match-abbruch-freilos-handling.md`. |
| UI consolidation of historically grown screens | Backlog seed; see `.planning/seeds/ui-consolidation-historically-grown-screens.md`. |
| CC-side admin UI changes | Carambus cannot modify the ClubCloud admin surface; all integration flows through the public CC API. |
| Non-CC tournament organizations | v7.1 scope is explicitly the CC integration; other upload targets (if any) are out of scope. |

## Open questions (resolve during v7.1 discuss-phase)

1. **CC API rate limits** — what are they, and do the finalization /
   player-creation flows need batching or retry logic? **[TBD: v7.1]**
2. **Error semantics** — if Endrangliste upload fails mid-transmission,
   what is the reconciliation story? **[TBD: v7.1]**
3. **Test environment** — is there a CC sandbox we can point automated
   tests at, or do we need VCR cassettes only? **[TBD: v7.1]**
4. **Impact on v7.0 upload model** — the "auto_upload_to_cc" checkbox
   was deliberately left alone in Phase 36b (FIX-02 closed as no-op)
   because the code and docs already agreed. Does v7.1 change that? If
   finalization is automated, does the per-game upload flow change too?
   **[TBD: v7.1]**
5. **Rollback** — if v7.1 ships a broken Endrangliste calculation, how
   does the Turnierleiter fall back to manual CC entry? **[TBD: v7.1]**
